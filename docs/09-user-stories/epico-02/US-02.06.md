## História de usuário

**Como** desenvolvedor novo que acabou de clonar o repositório,
**quero** encontrar no `CONTRIBUTING.md` instruções claras de como configurar `JWT_SECRET` e `JWT_SESSION_TIME` antes de tentar rodar a aplicação,
**para que** eu não precise ler o código-fonte ou perguntar no time só para descobrir por que `./gradlew bootRun` falha na primeira tentativa.

---

## Problema atual

O `CONTRIBUTING.md` (50 linhas) documenta fluxo de trabalho, Conventional Commits e o branch `bleeding` — mas não menciona nenhuma variável de ambiente. Um desenvolvedor novo só vai descobrir a necessidade de `JWT_SECRET` ao rodar a aplicação e ver a mensagem de erro do `JwtSecretFailureAnalyzer` (US-02.02). Isso funciona, mas obriga a passar pelo erro primeiro — o `CONTRIBUTING.md` deveria evitar esse passo.

## O que esta história entrega

Uma nova seção **"Configuração de ambiente"** no `CONTRIBUTING.md`, entre "Convenções" e "Branch `bleeding`", cobrindo: as duas variáveis JWT, como gerar `JWT_SECRET` com segurança, os valores de `JWT_SESSION_TIME` por perfil, e como ativar `SPRING_PROFILES_ACTIVE`.

## Conteúdo a adicionar ao `CONTRIBUTING.md`

Inserir após a seção `## Convenções` (linha 24, antes do separador `---`):

```markdown
---

## Configuração de ambiente

Antes de rodar `./gradlew bootRun` ou os testes, configure as variáveis de ambiente obrigatórias.

### Variáveis JWT (obrigatórias)

| Variável | Obrigatória? | Padrão se ausente | Referência |
|---|---|---|---|
| `JWT_SECRET` | **Sim** — a aplicação não inicia sem ela | Nenhum (fail-fast) | ADR-006 |
| `JWT_SESSION_TIME` | Não | `86400` (24h) | ADR-006 |

**Como gerar `JWT_SECRET` de forma segura:**

```bash
export JWT_SECRET=$(openssl rand -base64 64)
```

Nunca use valores fracos, placeholders conhecidos ou o mesmo valor entre ambientes diferentes.

**Configuração rápida:**

```bash
cp .env.example .env
# edite .env e preencha JWT_SECRET com o comando acima
source .env
```

### Perfis Spring (`SPRING_PROFILES_ACTIVE`)

| Perfil | `JWT_SESSION_TIME` padrão | Quando usar |
|---|---|---|
| `dev` | 604800 (7 dias) | Desenvolvimento local |
| `staging` | 86400 (24 horas) | Homologação |
| `prod` | 3600 (1 hora) | Produção |

```bash
export SPRING_PROFILES_ACTIVE=dev
```

Veja `.env.example` para o template completo e `docs/06-architecture-decisions.md` (ADR-006) para o racional completo da decisão.
```
