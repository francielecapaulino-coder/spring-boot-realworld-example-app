## História de usuário

**Como** operador de infraestrutura que gerencia múltiplos ambientes,
**quero** que cada ambiente (dev, staging, prod) tenha um perfil Spring com configurações específicas de segurança, logging e documentação de API —
**para que** o tempo de sessão JWT, a verbosidade de logs e a exposição do Swagger sejam automaticamente ajustados ao ativar o perfil, sem necessidade de alterar código ou configuração manual.

---

## Decisão arquitetural de referência

> **ADR-006** (`06-architecture-decisions.md` v1.0) define:

| Ambiente | `JWT_SESSION_TIME` default | Justificativa |
|---|---|---|
| dev | 604800 (7 dias) | Menos fricção para desenvolvimento — token não expira durante a semana |
| staging | 86400 (24 horas) | Próximo de produção — testa expiração real |
| prod | 3600 (1 hora) | Mais seguro — superfície de ataque reduzida em caso de token comprometido |

---

## Princípio de mínima duplicação

Os perfis **NÃO** duplicam `application.properties`. Cada arquivo contém **APENAS** o que é diferente por ambiente:

| Propriedade | Nos perfis? | Motivo |
|---|---|---|
| `jwt.secret` | ❌ NÃO | Mesmo valor para todos os ambientes — herda do base `${JWT_SECRET}` |
| `jwt.sessionTime` | ✅ SIM | Default diferente por ambiente (ADR-006) |
| `logging.level.*` | ✅ SIM | Nível diferente por ambiente |
| `springdoc.*` | ✅ Apenas prod | Swagger desabilitado em produção (antecipa EPIC-11) |

> **Nota sobre ADR-006:** o exemplo do ADR (linhas 628-637) inclui `jwt.secret` nos perfis para fins ilustrativos. A implementação omite intencionalmente — o valor é idêntico em todos os ambientes, portanto vive apenas no `application.properties` base. Spring Boot resolve por herança de perfil.

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

1. Cria `application-dev.yml` com JWT 604800, logging DEBUG
2. Cria `application-staging.yml` com JWT 86400, logging INFO
3. Cria `application-prod.yml` com JWT 3600, logging WARN, Swagger desabilitado
4. Remove os `.properties` de perfil antigos (evita ambiguidade de formato)
5. Atualiza `.env.example`: `SPRING_PROFILES_ACTIVE` com perfis disponíveis

### ❌ O que esta história NÃO faz

| O que | Onde será feito |
|---|---|
| Alterar `application.properties` base | US-02.01 (já feito) |
| Alterar `DefaultJwtService.java` | EPIC-04 |
| Configurar scan de secrets no CI | US-02.05 |
| Desabilitar Swagger completamente | EPIC-11 (configuração já preparada em prod) |
| Configurar JPA por perfil | EPIC-05 (placeholders comentados já nos arquivos) |
