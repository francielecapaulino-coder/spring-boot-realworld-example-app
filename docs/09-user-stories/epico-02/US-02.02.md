## História de usuário

**Como** desenvolvedor do time que está configurando o ambiente local,  
**quero** receber uma mensagem de erro clara e acionável quando esqueço de definir `JWT_SECRET` —  
**para que** eu saiba exatamente o que fazer para corrigir o problema em segundos, sem precisar consultar documentação ou depurar o stack trace do Spring.

## Problema atual (após US-02.01)

Com `jwt.secret=${JWT_SECRET}` no `application.properties`, se `JWT_SECRET` não estiver definida, o Spring Boot lança:

````
***************************
APPLICATION FAILED TO START
***************************

Description:

Failed to bind properties under 'jwt.secret' to java.lang.String:

    Reason: Could not resolve placeholder 'JWT_SECRET' in value "${JWT_SECRET}"

Action:

Ensure that your configuration does not have unresolvable placeholders
````

**O problema:** esta mensagem não diz o que é `JWT_SECRET`, como gerar um valor seguro, onde configurar, nem referencia os documentos do projeto. Um desenvolvedor novo perde minutos ou horas.

## Contexto técnico completo

### Como funciona o FailureAnalyzer no Spring Boot 4

O Spring Boot tem um mecanismo de `FailureAnalyzer` que intercepta exceções durante o startup e as transforma em mensagens amigáveis antes de exibir no console. É o mecanismo que produz as mensagens `APPLICATION FAILED TO START` com `Description` e `Action`.

**Fluxo de execução:**

````
Spring Boot inicia
      ↓
Tenta resolver ${JWT_SECRET} no application.properties
      ↓
JWT_SECRET ausente → lança IllegalArgumentException
      ↓
SpringApplication captura a exceção
      ↓
Executa todos os FailureAnalyzers registrados
      ↓
JwtSecretFailureAnalyzer reconhece a exceção (verifica se mensagem contém "JWT_SECRET")
      ↓
Retorna FailureAnalysis com Description + Action customizados
      ↓
Spring Boot exibe a mensagem formatada e encerra
````

### Por que `FailureAnalyzer` e não `EnvironmentPostProcessor`

| Abordagem | Quando executa | Problema |
|---|---|---|
| `@PostConstruct` em DefaultJwtService | Quando o bean é criado | Spring já falhou antes de criar o bean |
| `ApplicationRunner` | Após o contexto subir | Contexto não sobe sem JWT_SECRET |
| `EnvironmentPostProcessor` | Antes de resolver placeholders | Funciona, mas mais complexo de registrar e testar |
| **`FailureAnalyzer`** ✅ | Quando o startup falha | Idiomático no Spring Boot 4, simples, testável |

O `FailureAnalyzer` é o padrão estabelecido pelo Spring Boot para transformar exceções de startup em mensagens acionáveis. É exatamente como o próprio Spring Boot formata os erros de porta já em uso, banco de dados indisponível, etc.

### Passo 2 — Criar `JwtSecretFailureAnalyzer.java`

````java
package io.spring.infrastructure.config;

import org.springframework.boot.diagnostics.AbstractFailureAnalyzer;
import org.springframework.boot.diagnostics.FailureAnalysis;

/**
 * FailureAnalyzer que intercepta a falha de startup causada pela ausência
 * de JWT_SECRET e exibe uma mensagem clara e acionável.
 *
 * Motivação: ADR-006 — JWT_SECRET é obrigatória sem fallback (fail-fast).
 * Sem esta classe, o Spring exibe apenas "Could not resolve placeholder 'JWT_SECRET'"
 * o que não orienta o desenvolvedor sobre como corrigir o problema.
 *
 * @see docs/06-architecture-decisions.md ADR-006
 */
public class JwtSecretFailureAnalyzer
        extends AbstractFailureAnalyzer<IllegalArgumentException> {

    private static final String JWT_SECRET_PLACEHOLDER = "JWT_SECRET";

    @Override
    protected FailureAnalysis analyze(Throwable rootFailure,
                                      IllegalArgumentException cause) {

        if (!isJwtSecretMissing(cause)) {
            return null; // não é nossa exceção — deixar outros analyzers tratarem
        }

        String description = """
                A variável de ambiente JWT_SECRET não está definida.
                JWT_SECRET é obrigatória para assinar tokens JWT (algoritmo HS512).
                Sem ela, nenhuma autenticação é possível e a aplicação não pode iniciar.""";

        String action = """
                1. Gere um valor seguro:
                   openssl rand -base64 64

                2. Exporte a variável no terminal atual:
                   export JWT_SECRET=<valor-gerado>

                3. Para desenvolvimento persistente, adicione ao seu .env local:
                   JWT_SECRET=<valor-gerado>
                   (o arquivo .env NÃO deve ser commitado — use .env.example como referência)

                Referência: docs/06-architecture-decisions.md — ADR-006""";

        return new FailureAnalysis(description, action, cause);
    }

    private boolean isJwtSecretMissing(IllegalArgumentException cause) {
        String message = cause.getMessage();
        return message != null && message.contains(JWT_SECRET_PLACEHOLDER);
    }
}
````

### Passo 4 — Criar `JwtSecretFailureAnalyzerTest.java`

````java
package io.spring.infrastructure.config;

import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;
import org.springframework.boot.diagnostics.FailureAnalysis;

import static org.assertj.core.api.Assertions.assertThat;

@DisplayName("JwtSecretFailureAnalyzer")
class JwtSecretFailureAnalyzerTest {

    private final JwtSecretFailureAnalyzer analyzer = new JwtSecretFailureAnalyzer();

    @Test
    @DisplayName("deve retornar FailureAnalysis quando JWT_SECRET está ausente")
    void deveRetornarAnalysisQuandoJwtSecretAusente() {
        // dado
        IllegalArgumentException cause = new IllegalArgumentException(
                "Could not resolve placeholder 'JWT_SECRET' in value \"${JWT_SECRET}\""
        );

        // quando
        FailureAnalysis analysis = analyzer.analyze(new RuntimeException(cause), cause);

        // então
        assertThat(analysis).isNotNull();
        assertThat(analysis.getDescription())
                .contains("JWT_SECRET")
                .contains("obrigatória")
                .contains("tokens JWT");
        assertThat(analysis.getAction())
                .contains("openssl rand -base64 64")
                .contains("export JWT_SECRET")
                .contains("ADR-006");
    }

    @Test
    @DisplayName("deve retornar null para exceção não relacionada a JWT_SECRET")
    void deveRetornarNullParaExcecaoNaoRelacionada() {
        // dado
        IllegalArgumentException cause = new IllegalArgumentException(
                "Could not resolve placeholder 'OUTRO_PLACEHOLDER' in value \"${OUTRO}\""
        );

        // quando
        FailureAnalysis analysis = analyzer.analyze(new RuntimeException(cause), cause);

        // então
        assertThat(analysis).isNull();
    }

    @Test
    @DisplayName("deve retornar null para exceção com mensagem nula")
    void deveRetornarNullParaMensagemNula() {
        // dado
        IllegalArgumentException cause = new IllegalArgumentException((String) null);

        // quando
        FailureAnalysis analysis = analyzer.analyze(new RuntimeException(cause), cause);

        // então
        assertThat(analysis).isNull();
    }

    @Test
    @DisplayName("description deve mencionar que a aplicação não pode iniciar")
    void descriptionDeveMencionarQueAplicacaoNaoPodeIniciar() {
        // dado
        IllegalArgumentException cause = new IllegalArgumentException(
                "Could not resolve placeholder 'JWT_SECRET' in value \"${JWT_SECRET}\""
        );

        // quando
        FailureAnalysis analysis = analyzer.analyze(new RuntimeException(cause), cause);

        // então
        assertThat(analysis).isNotNull();
        assertThat(analysis.getDescription())
                .contains("aplicação não pode iniciar");
    }

    @Test
    @DisplayName("action deve referenciar o arquivo .env.example")
    void actionDeveReferenciarEnvExample() {
        // dado
        IllegalArgumentException cause = new IllegalArgumentException(
                "Could not resolve placeholder 'JWT_SECRET' in value \"${JWT_SECRET}\""
        );

        // quando
        FailureAnalysis analysis = analyzer.analyze(new RuntimeException(cause), cause);

        // então
        assertThat(analysis).isNotNull();
        assertThat(analysis.getAction())
                .contains(".env.example");
    }
}
````

````
import org.springframework.boot.diagnostics.AbstractFailureAnalyzer;
import org.springframework.boot.diagnostics.FailureAnalysis;

public class JwtSecretFailureAnalyzer
        extends AbstractFailureAnalyzer<IllegalArgumentException> {

    private static final String JWT_SECRET_PLACEHOLDER = "JWT_SECRET";

    @Override
    protected FailureAnalysis analyze(Throwable rootFailure,
                                      IllegalArgumentException cause) {
        if (!isJwtSecretMissing(cause)) {
            return null;
        }
        // Usar Java 25 text blocks para description e action
        // description: o que é o problema e por que a app não pode iniciar
        // action: 3 passos (openssl, export, .env local) + referência ADR-006
        return new FailureAnalysis(description, action, cause);
    }

    private boolean isJwtSecretMissing(IllegalArgumentException cause) {
        String message = cause.getMessage();
        return message != null && message.contains(JWT_SECRET_PLACEHOLDER);
    }
}
````
