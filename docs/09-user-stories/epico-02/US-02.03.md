## História de usuário

**Como** novo desenvolvedor que acaba de clonar o repositório,
**quero** encontrar um arquivo `.env.example` na raiz com todas as variáveis de ambiente necessárias, seus valores padrão, comentários explicativos e comandos de geração —
**para que** eu consiga configurar o ambiente local em menos de 5 minutos, sem precisar ler código-fonte ou perguntar a outro membro do time.

## Contexto técnico

### Por que `.env.example` é necessário

Após US-02.01, o `application.properties` contém:

````properties
jwt.secret=${JWT_SECRET}
jwt.sessionTime=${JWT_SESSION_TIME:86400}
````

Sem `.env.example`, um novo desenvolvedor:
1. Clona o repositório
2. Executa `./gradlew bootRun`
3. Recebe: `APPLICATION FAILED TO START — Could not resolve placeholder 'JWT_SECRET'`
4. Não sabe o que configurar, que formato usar, nem como gerar um valor seguro

O `JwtSecretFailureAnalyzer` (US-02.02) mitiga parcialmente com uma mensagem clara, mas `.env.example` resolve o problema **antes** da primeira tentativa.

### Decisão arquitetural de referência

> **ADR-006** (`06-architecture-decisions.md` v1.0) define:
> - `JWT_SECRET`: obrigatória — sem valor padrão. Algoritmo HS512, mínimo 64 bytes.
> - `JWT_SESSION_TIME`: opcional — fallback 86400 (24h). Valores por perfil: dev=604800, staging=86400, prod=3600.
> - `.env.example` deve documentar ambas as variáveis com exemplos de valores por ambiente.

### Convenção `.env.example` vs `.env`

| Arquivo | Propósito | Git |
|---|---|---|
| `.env.example` | Template com placeholders e documentação | ✅ Commitado |
| `.env` | Valores reais do ambiente local | ❌ No `.gitignore` |

O `.env` já consta no `.gitignore` (adicionado junto com esta história).

1. Cria `.env.example` na raiz com 4 seções:
   - **Seção 1 — JWT (EPIC-02):** `JWT_SECRET` + `JWT_SESSION_TIME`

## Conteúdo exato do `.env.example`

````bash
# ============================================================================
# .env.example — RealWorld Platform Modernization
# ============================================================================
#
# Como usar:
#   1. cp .env.example .env
#   2. Preencha os valores (veja instrucoes abaixo de cada variavel)
#   3. source .env                          (carrega no shell atual)
#
# Opcao Spring Boot 4 nativa (sem source necessario):
#   Adicione ao application.properties:
#     spring.config.import=optional:file:.env[properties]
#
# Docker Compose (EPIC-03):
#   Carrega .env automaticamente — basta ter o arquivo na raiz.
#
# Protecao complementar:
#   US-02.05 configura truffleHog no CI para detectar commits de secrets.
#
# Referencias:
#   - ADR-006: docs/06-architecture-decisions.md
#   - Definition of Ready: docs/process/definition-of-ready.md
# ============================================================================


# ============================================================================
# SECAO 1 — SEGURANCA — JWT (EPIC-02)
# ============================================================================

# JWT_SECRET — OBRIGATORIA — a aplicacao NAO inicia sem este valor (fail-fast, ADR-006)
# Algoritmo: HS512 — minimo 64 bytes (512 bits)
# Como gerar JWT_SECRET:
#   openssl rand -base64 64
# NUNCA use valores fracos, senhas comuns ou placeholders conhecidos
# NUNCA compartilhe ou commite o valor real de JWT_SECRET
JWT_SECRET=your-secret-here-generate-with-openssl-rand-base64-64

# OPCIONAL — padrao 86400 (24h) se nao definida (ADR-006)
# Valores por ambiente:
#   dev     = 604800  (7 dias)
#   staging = 86400   (24 horas)
#   prod    = 3600    (1 hora)
JWT_SESSION_TIME=86400


# ============================================================================
# SECAO 2 — PERFIL DA APLICACAO (EPIC-02)
# ============================================================================

# Perfis disponíveis: dev | staging | prod
# Descomente e ajuste conforme o ambiente:
# SPRING_PROFILES_ACTIVE=dev


# ============================================================================
# SECAO 3 — BANCO DE DADOS — PostgreSQL (EPIC-03)
# ============================================================================

# Descomente quando EPIC-03 estiver concluido
# POSTGRES_PASSWORD: gerar com openssl rand -base64 32
# POSTGRES_DB=realworld
# POSTGRES_USER=postgres
# POSTGRES_PASSWORD=your-postgres-password-here-generate-with-openssl-rand-base64-32
# POSTGRES_HOST=localhost
# POSTGRES_PORT=5432


# ============================================================================
# SECAO 4 — OBSERVABILIDADE — LGTM Stack (EPIC-10)
# ============================================================================

# Placeholder — variaveis de observabilidade serao adicionadas em EPIC-10 (Fase 4)


# ============================================================================
# Referencias
# ============================================================================
# ADR-006:             docs/06-architecture-decisions.md
# Definition of Ready: docs/process/definition-of-ready.md
# CONTRIBUTING.md:     disponivel apos US-02.06
````
