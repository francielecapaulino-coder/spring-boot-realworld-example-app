## História de usuário

**quero** escrever testes de integração para os endpoints de usuário,  
**para que** o contrato REST de autenticação seja validado contra banco real.

---

## Contexto

Controllers REST de usuário no projeto:
- `UsersApi.java` — `POST /users` (register), `POST /users/login`
- `CurrentUserApi.java` — `GET /user`, `PUT /user`
- `ProfileApi.java` — `GET /profiles/:username`, `POST /profiles/:username/follow`, `DELETE /profiles/:username/follow`

Testes existentes em `UsersApiTest.java`, `CurrentUserApiTest.java`, `ProfileApiTest.java` — verificar antes de criar.

---

## Escopo

* Completar ou criar testes `@SpringBootTest` + `MockMvc` + Testcontainers PostgreSQL para:
  - `POST /users` — sucesso (201), email duplicado (422), campos inválidos (422)
  - `POST /users/login` — sucesso (200), senha errada (401), usuário inexistente (401)
  - `GET /user` — com token válido (200), sem token (401)
  - `PUT /user` — atualização de campos (200), token inválido (401)
  - `GET /profiles/:username` — usuário existente (200), inexistente (404)
  - `POST /profiles/:username/follow` — seguir (200), sem token (401)
  - `DELETE /profiles/:username/follow` — deixar de seguir (200), sem token (401)
* Reutilizar `TestHelper` existente para criação de usuários de teste

## Fora de escopo

* Testes de Article, Comment, Tags — US-08.03 e US-08.04
* Testes GraphQL — US-08.05

## Critérios de aceite

* [ ] Todos os 7 endpoints de usuário/perfil têm ao menos 1 teste de sucesso e 1 de erro
* [ ] Testes de autenticação: com token válido, sem token, token inválido
* [ ] `./gradlew test` 100% verde com PostgreSQL real (Testcontainers)
* [ ] Nenhum teste usa H2

## Regras técnicas

* Base class deve estender `DbTestBase` ou classe equivalente com Testcontainers
* Usar `MockMvc` com `@AutoConfigureMockMvc` — não subir servidor completo desnecessariamente
* Não criar dados de teste fora de `@BeforeEach` — evitar estado compartilhado entre testes

## Dependências

## Resultado esperado

7 endpoints de usuário/perfil com contrato REST validado por testes de integração.
