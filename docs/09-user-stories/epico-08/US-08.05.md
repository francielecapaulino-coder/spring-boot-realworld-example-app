## História de usuário

**Como** desenvolvedor responsável pela qualidade,  
**quero** escrever testes de integração para as operações GraphQL,  
**para que** o contrato GraphQL seja validado contra banco real e não quebre silenciosamente.

---

## Contexto

O projeto usa DGS Framework 12.x (integra nativamente com Spring for GraphQL). As 18 operações GraphQL são:

**Queries (6):** `article`, `articles`, `me`, `feed`, `profile`, `tags`  
**Mutations (12):** `createUser`, `login`, `updateUser`, `followUser`, `unfollowUser`, `createArticle`, `updateArticle`, `favoriteArticle`, `unfavoriteArticle`, `deleteArticle`, `addComment`, `deleteComment`

---

## Escopo

* Criar testes `@SpringBootTest` + DGS Test client + Testcontainers PostgreSQL
* Cobrir ao menos 1 cenário de sucesso por operação (18 no total)
* Cobrir cenários de erro para mutations com autenticação: sem token → erro esperado
* Usar `DgsQueryExecutor` (DGS test utility) para executar queries e mutations programaticamente

## Fora de escopo

* Testes de performance GraphQL
* Introspection de schema completo — apenas operações funcionais

## Critérios de aceite

* [ ] 6 queries GraphQL com ao menos 1 teste de sucesso cada
* [ ] 12 mutations GraphQL com ao menos 1 teste de sucesso e 1 de autorização cada
* [ ] `./gradlew test` 100% verde com Testcontainers PostgreSQL
* [ ] Testes executam contra o schema GraphQL real (sem mocks de resolvers)

## Regras técnicas

* Usar `DgsQueryExecutor` injetado no teste — é a forma idiomática no DGS para testes de integração
* Não mockar os resolvers GraphQL — testes de integração devem ir até o banco
* Reutilizar `TestHelper` para setup de dados

## Dependências

* US-08.04 concluído
* EPIC-04 concluído (DGS 12.x funcional)

## Resultado esperado

18 operações GraphQL com contrato validado.
