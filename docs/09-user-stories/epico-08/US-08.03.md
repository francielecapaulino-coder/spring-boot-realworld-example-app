## Contexto

Controllers REST de artigo no projeto:
- `ArticlesApi.java` — `GET /articles` (listagem com filtros), `POST /articles`
- `ArticleApi.java` — `GET /articles/:slug`, `PUT /articles/:slug`, `DELETE /articles/:slug`
- `ArticleFavoriteApi.java` — `POST /articles/:slug/favorite`, `DELETE /articles/:slug/favorite`

Testes existentes: `ArticlesApiTest.java`, `ArticleApiTest.java`, `ArticleFavoriteApiTest.java`, `ListArticleApiTest.java` — verificar antes de criar.

---

## Escopo

* Completar ou criar testes de integração para:
  - `GET /articles` — listagem com filtros `tag`, `author`, `favorited`, paginação `limit`/`offset`
  - `POST /articles` — criação com campos válidos (201), sem token (401), campos inválidos (422)
  - `GET /articles/:slug` — artigo existente (200), slug inexistente (404)
  - `PUT /articles/:slug` — atualização pelo autor (200), por outro usuário (403), sem token (401)
  - `DELETE /articles/:slug` — remoção pelo autor (200), por outro usuário (403)
  - `GET /articles/feed` — feed do usuário autenticado (200), sem token (401)
  - `POST/DELETE /articles/:slug/favorite` — favoritar/desfavoritar (200), sem token (401)

## Fora de escopo

* Soft delete — coberto em EPIC-12
* Reading time — coberto em EPIC-13

## Critérios de aceite

* [ ] 7 endpoints de artigo com ao menos 1 teste de sucesso e 1 de erro cada

* [ ] `./gradlew test` 100% verde com Testcontainers PostgreSQL
* [ ] Filtros de listagem (`tag`, `author`, `favorited`) testados
