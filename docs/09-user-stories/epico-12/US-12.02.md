## História de usuário

**Como** operador da plataforma,  
**quero** que a exclusão de um comentário marque o registro como `is_deleted = true` em vez de removê-lo permanentemente,  
**para que** o conteúdo possa ser auditado e o comportamento visível para o usuário permaneça idêntico.

---

## Contexto

Após US-12.01 (artigos), o mesmo padrão deve ser aplicado à entidade `Comment`. `DELETE /articles/:slug/comments/:id` atualmente remove o registro permanentemente. Esta história adiciona coluna `is_deleted` em `comments` via Flyway, anota `Comment` com `@SQLRestriction` e altera o serviço de deleção para soft delete. O padrão é idêntico ao de artigos — apenas a entidade e a migration mudam.

---

## Escopo

* Criar migration Flyway: `ALTER TABLE comments ADD COLUMN is_deleted BOOLEAN NOT NULL DEFAULT false`
* Anotar entidade `Comment` com `@SQLRestriction("is_deleted = false")`
* Modificar `CommentRepository` para executar `UPDATE SET is_deleted = true` no delete
* Garantir que `GET /articles/:slug/comments` **não retorna** comentários com `is_deleted = true`
* Garantir que `DELETE /articles/:slug/comments/:id` retorna `200` (comportamento externo inalterado)
* Adicionar testes de integração: deletar comentário → confirmar que não aparece na listagem

## Fora de escopo

* Restauração ou listagem de comentários deletados
* Soft delete em outras entidades (follow, favorite, user)
* Mudança em autorização do DELETE

## Critérios de aceite

* [ ] Migration Flyway executada sem erro: coluna `is_deleted` existe em `comments` com default `false`
* [ ] `DELETE /articles/:slug/comments/:id` retorna `200` e comentário não aparece em `GET /articles/:slug/comments`
* [ ] Testes de integração cobrem: delete + listagem do artigo
* [ ] `./gradlew test` 100% verde

## Regras técnicas

* Mesma abordagem de US-12.01: `@SQLRestriction`, migration versionada, campo nunca exposto na API
* Migration: `V<n+1>__add_is_deleted_to_comments.sql`

## Dependências

## Resultado esperado

Comentários excluídos permanecem no banco invisíveis para o consumidor da API, com auditoria habilitada.
