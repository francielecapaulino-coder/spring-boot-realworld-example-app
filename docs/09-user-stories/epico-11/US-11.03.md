## História de usuário

## Contexto

Controllers alvo:
- `ArticlesApi.java` — `GET /articles`, `POST /articles`, `GET /articles/feed` (3 endpoints)
- `ArticleApi.java` — `GET /articles/:slug`, `PUT /articles/:slug`, `DELETE /articles/:slug` (3 endpoints)
- `ArticleFavoriteApi.java` — `POST /articles/:slug/favorite`, `DELETE /articles/:slug/favorite` (2 endpoints)
- `CommentsApi.java` — `GET /articles/:slug/comments`, `POST /articles/:slug/comments`, `DELETE /articles/:slug/comments/:id` (3 endpoints)
- `TagsApi.java` — `GET /tags` (1 endpoint)

Total: 12 endpoints + 7 de US-11.02 = 19 endpoints.

---

## Escopo

* Adicionar `@Operation(summary, description)` em cada método handler dos 5 controllers
* Adicionar `@ApiResponse` para cada código HTTP documentado (200, 201, 401, 403, 404, 422)
* Adicionar `@Tag(name = "Articles")`, `@Tag(name = "Comments")`, `@Tag(name = "Tags")` nos controllers
* Documentar parâmetros de query (`tag`, `author`, `favorited`, `limit`, `offset`) com `@Parameter`
* Verificar que os 12 endpoints aparecem corretamente no Swagger UI

## Fora de escopo

* Schemas customizados de request/response — gerados automaticamente
* Documentação GraphQL — fora do escopo do mandato (OpenAPI é para REST)

## Critérios de aceite

* [ ] 12 endpoints de artigo/comentário/tags com `@Operation` e ao menos 1 `@ApiResponse`
* [ ] Parâmetros de query de `GET /articles` documentados com `@Parameter`
* [ ] Swagger UI exibe 19 endpoints no total, agrupados por tags
* [ ] `./gradlew test` 100% verde
* [ ] `/v3/api-docs` contém os 19 endpoints documentados

## Regras técnicas

* Mesmo padrão de anotação de US-11.02 — consistência é obrigatória
* `@Parameter(description = "...", example = "...")` para query params com valores de exemplo
* Não alterar lógica de negócio

## Dependências

* US-11.02 concluído

## Resultado esperado

19 endpoints REST completamente documentados no Swagger UI.
