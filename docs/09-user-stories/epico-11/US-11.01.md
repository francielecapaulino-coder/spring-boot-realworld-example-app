## História de usuário

**Como** consumidor/auditor da API REST,  
**quero** ver o contrato atual em formato OpenAPI/Swagger exportado a partir do código-fonte,  
**para que** exista documentação viva, sincronizada com os campos e endpoints reais do sistema.

---

## Contexto

Spring Boot 4 com SpringDoc OpenAPI gera OpenAPI 3 automaticamente via anotações nos controllers. O projeto já tem endpoints REST completos. Precisamos configurar springdoc-openapi-ui para expor `/v3/api-docs` e `/swagger-ui.html`. Também gerar arquivo `openapi.json` no `build` para versionamento e facilitar consumo externo (como Pact/Playwright). Esta história foca em gerar o contrato existente; adições de campos (reading time, soft delete) virão em histórias futuras (US-11.02+).

---

## Escopo

* Adicionar dependência `org.springdoc:springdoc-openapi-starter-webmvc-ui` ao `build.gradle`
* Configurar `springdoc.api-docs.path=/v3/api-docs`
* Configurar `springdoc.swagger-ui.path=/swagger-ui.html`
* Adicionar infos como title, version, description, contact via `@OpenAPIDefinition` no package de API
* Gerar arquivo estático `openapi.json` no build (tarefa customizada ou springdoc generate)
* Commitar `openapi.json` no repo para versionamento (ou gerar em CI)

## Fora de escopo

* Inclusão de novos campos específicos ao negócio (reading time, soft delete)
* Autenticação na UI do Swagger
* Validação automática de contract testing com a spec

## Critérios de aceite

* [ ] `./gradlew build` gera `openapi.json` na raiz ou `src/main/resources/static/docs`
* [ ] Acesso `http://localhost:8080/swagger-ui.html` funciona e lista endpoints
* [ ] `/v3/api-docs` retorna JSON com paths para users, articles, profiles, comments, tags
* [ ] Generate JSON inclui schemas de DTOs com tipos corretos (Instant, não Joda)
* [ ] Não há enums faltando ou nomes incorretos

## Regras técnicas

* SpringDoc OpenAPI 2.x é compatível com Spring Boot 4 e usa Jakarta
* Classes Lombok (entidades) ainda podem gerar schema
* Se houver campos `@JsonIgnore` nos DTOs, ajustar com `@Schema(hidden=true)` se necessário

## Dependências

* EPIC-04 (runtime e Jakarta)
* EPIC-08 (repositório integrados para validação)

## Resultado esperado

OpenAPI spec v3 gerado e disponível via UI, servindo como fonte única do contrato público da API.
