---

## Contexto

Controllers alvo:
- `UsersApi.java` — `POST /users` (register), `POST /users/login`
- `CurrentUserApi.java` — `GET /user`, `PUT /user`
- `ProfileApi.java` — `GET /profiles/:username`, `POST /profiles/:username/follow`, `DELETE /profiles/:username/follow`

**Padrão de anotação (springdoc-openapi 2.x):**
```java
@Operation(summary = "Register a new user", description = "Creates a user account and returns a JWT token")
@ApiResponse(responseCode = "201", description = "User created")
@ApiResponse(responseCode = "422", description = "Validation error — email or username already taken")
```

---

## Escopo

* Adicionar `@Operation(summary, description)` em cada método handler dos 3 controllers
* Adicionar `@ApiResponse` para cada código HTTP documentado (201, 200, 401, 403, 422)
* Adicionar `@Tag(name = "Users")` e `@Tag(name = "Profiles")` nos controllers
* Verificar que os 7 endpoints aparecem corretamente no Swagger UI em `http://localhost:8080/swagger-ui.html`

## Fora de escopo

* Schemas de request/response — gerados automaticamente pelo springdoc a partir dos tipos Java
* Article, Comment, Tags — US-11.03

## Critérios de aceite

* [ ] 7 endpoints de usuário/perfil com `@Operation` e ao menos 1 `@ApiResponse`
* [ ] Swagger UI exibe os 7 endpoints agrupados em tags "Users" e "Profiles"
* [ ] `./gradlew test` 100% verde (anotações não quebram testes existentes)
* [ ] `/v3/api-docs` contém os 7 endpoints documentados

## Regras técnicas

* Anotações adicionadas nos métodos handler — não nas interfaces
* Usar springdoc 2.x (`io.swagger.v3.oas.annotations.*`) — não Springfox (`io.swagger.annotations.*`)
* Não alterar lógica de negócio — somente anotações

## Dependências

* US-11.01 concluído (springdoc no classpath, Swagger UI funcionando)

## Resultado esperado

7 endpoints de usuário/perfil documentados no Swagger UI.
