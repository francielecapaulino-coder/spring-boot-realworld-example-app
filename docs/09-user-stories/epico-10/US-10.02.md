## História de usuário

**Como** operador da plataforma,  
**quero** que cada endpoint REST incremente um contador Micrometer com sua identificação,  
**para que** o Prometheus possa coletar e o Grafana possa exibir o volume de chamadas por endpoint.

---

## Contexto

O projeto tem 19 endpoints REST em 7 controllers:
- `UsersApi.java` (2 endpoints), `CurrentUserApi.java` (2), `ProfileApi.java` (3)
- `ArticlesApi.java` (3), `ArticleApi.java` (3), `ArticleFavoriteApi.java` (2)
- `CommentsApi.java` (3), `TagsApi.java` (1)

**Padrão do contador (do INI-10):**
```java
Counter.builder("api.requests.total")
    .tag("endpoint", "list_articles")
    .tag("method", "GET")
    .tag("path", "/articles")
    .register(meterRegistry)
    .increment();
```

---

## Escopo

* Injetar `MeterRegistry` em cada controller REST via construtor
* Adicionar `Counter` com tags `endpoint`, `method`, `path` em cada método handler
* Nomes de métrica consistentes: `api.requests.total` para todos os endpoints
* Verificar que `/actuator/prometheus` exibe as métricas após chamada aos endpoints

## Fora de escopo

* Contadores de erro (4xx, 5xx) — o Spring Boot Actuator já fornece via `http.server.requests`
* Histogramas de latência — o Micrometer auto-instrumenta via `http.server.requests`
* Contadores em endpoints GraphQL — fora do escopo do mandato

## Critérios de aceite

* [ ] 19 endpoints REST com contador `api.requests.total` + tags corretas
* [ ] `GET /actuator/prometheus` exibe `api_requests_total{endpoint="...",method="...",path="..."}` para cada endpoint chamado
* [ ] `./gradlew test` 100% verde
* [ ] Nenhuma regressão em endpoints existentes

## Regras técnicas

* Injetar `MeterRegistry` via construtor (não via `@Autowired` em campo)
* Nomenclatura de tag `endpoint`: snake_case, ex: `list_articles`, `get_article`, `create_user`
* O contador deve ser criado no construtor e incrementado no método handler (não a cada chamada de `Counter.builder()`)

## Dependências

* US-10.01 concluído (Prometheus auto-configuration ativa)

## Resultado esperado

19 endpoints com métricas de chamada disponíveis no Prometheus.
