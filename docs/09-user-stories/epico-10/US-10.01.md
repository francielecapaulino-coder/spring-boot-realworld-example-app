## História de usuário

**Como** engenheiro de SRE responsável pela observabilidade do sistema,  
**quero** levantar a stack LGTM (Loki/Grafana Tempo/Prometheus) via Docker Compose e expormétricas básicas via micrometer,  
**para que** cada endpoint exporte contadores e consigamos validar nos logs de startup/shutdown e traces habilitados.

---

## Contexto

LGTM é uma stack de observabilidade open-source leve: Loki logs, Grafana dashboards, Tempo traces, Prometheus metrics. Com Spring Boot 4 + micrometer já vem com Prometheus endpoint. Precisamos contratos Compose para Loki/Prometheus/Tempo/Grafana, e expor métricas pelo `/actuator/prometheus`. Não há UI centralizada, apenas validate via script在后. Traces podem ser exportados para Tempo via OpenTelemetry exporter.

---

## Escopo

* Adicionar contêineres `loki`, `prometheus`, `tempo`, `grafana` ao `docker-compose.yml`
* Expor HTTP scrape do Prometheus na porta 9090
* Configurar Loki para receber logs do app via file driver; app deve usar pattern JSON
* Adicionar micrometer Prometheus auto-configuration (já em Spring Boot 4)
* Expor `management.endpoints.web.exposure.include=prometheus,health,info,metrics`
* Adicionar `management.tracing.enabled=true` e `management.otlp.tracing.endpoint=http://tempo:4318/v1/traces`
* Criar dashboards básicos: contadores por endpoint e uptime
* Habilitar `logging.loggly.enabled=false` e `logging.pattern.console=%d{yyyy-MM-dd HH:mm:ss} ...` JSON

## Fora de escopo

* Custom alertas no Prometheus ou notificações
* Métricas de business (reading time, soft delete ainda não existem)
* Autenticação no Grafana
* Script Python de validação (US-10.03)

## Critérios de aceite

* [ ] `docker-compose up -d` sobe stack Loki/Grafana/Tempo/Prometheus + app
* [ ] Prometheus scrape `/actuator/prometheus` e mostra métricas JVM
* [ ] Grafana dash mostra contadores de HTTP requests
* [ ] Loki coleta logs em formato JSON de startup/shutdown
* [ ] Traces são exportados para Tempo e se pode visualizar pelo Grafana

## Regras técnicas

* Version pins de imagens estável ( Loki latest ou 3.x, Tempo latest ou 2.x, Grafana latest )
* Não expor ports públicos desnecessários no ambiente dev (acesse em localhost:3000 etc.)
