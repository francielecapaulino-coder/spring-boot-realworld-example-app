## História de usuário

**Como** operador da plataforma,  
**quero** um dashboard Grafana com métricas por endpoint, logs e traces,  
**para que** o estado da aplicação seja observável em um único lugar.

---

## Contexto

O `docker-compose.yml` já deve ter Prometheus, Grafana e Loki após US-10.01. O diretório `grafana/provisioning/datasources/` já existe no projeto — verificar o conteúdo antes de implementar.

**Painéis mínimos do dashboard:**

| Painel | Fonte | Query |
|---|---|---|
| Chamadas por endpoint (série temporal) | Prometheus | `rate(api_requests_total[5m])` por `endpoint` |
| Taxa de erros HTTP | Prometheus | `rate(http_server_requests_seconds_count{status=~"4..|5.."}[5m])` |
| Logs da aplicação | Loki | `{app="realworld"}` |
| Uptime | Prometheus | `process_uptime_seconds` |

---

## Escopo

* Criar arquivo de provisioning do dashboard em `grafana/provisioning/dashboards/realworld.json`
* Configurar datasources Prometheus e Loki via provisioning (YAML em `grafana/provisioning/datasources/`)
* Dashboard com ao menos 4 painéis (listados acima)
* Grafana acessível em `http://localhost:3000` após `docker-compose up`

## Fora de escopo

* Alertas Grafana (future)
* Traces Tempo (configuração básica já deve estar em US-10.01 — sem dashboard dedicado nesta história)

## Critérios de aceite

* [ ] `docker-compose up` sobe Grafana em `http://localhost:3000`
* [ ] Dashboard "RealWorld" disponível sem configuração manual
* [ ] Painel de chamadas por endpoint exibe dados após chamadas à API
* [ ] Painel de logs Loki exibe logs JSON da aplicação

## Regras técnicas

* Dashboard deve ser provisionado via arquivo JSON — não configurado manualmente no Grafana
* Credenciais padrão Grafana: `admin/admin` — não alterar para este ambiente de desenvolvimento

## Dependências

* US-10.01, US-10.02 e US-10.03 concluídos

## Resultado esperado

Dashboard Grafana funcional com métricas, logs e uptime da aplicação.
