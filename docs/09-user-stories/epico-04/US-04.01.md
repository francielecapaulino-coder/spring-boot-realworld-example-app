## História de usuário

**Como** Tech Lead que precisa decidir a estratégia de GraphQL antes do upgrade do framework,
**quero** que exista um ADR formal comparando manter o Netflix DGS Framework (versão 10.x) versus migrar para o Spring for GraphQL nativo —
**para que** a equipe inicie o EPIC-04 com a decisão registrada, rastreável e sem retrabalho de resolvers.

---

## Contexto

O projeto usa o **Netflix DGS Framework 4.9.21** para expor a API GraphQL. Com o upgrade para Spring Boot 4.0.3 (US-04.03), é necessário decidir entre:

- **Opção A** — atualizar o DGS para a versão **10.x** (reimplementada sobre o Spring for GraphQL internamente; anotações `@DgsComponent`/`@DgsQuery`/`@DgsMutation` inalteradas);
- **Opção B** — migrar para **Spring for GraphQL nativo** (reescrita completa dos resolvers: `@DgsQuery` → `@QueryMapping`, etc.).

A decisão já está consolidada no `docs/06-architecture-decisions.md` como **ADR-001 — Opção A (DGS 10.x)**. Esta história garante que o ADR esteja **completo, aprovado e referenciado** antes de qualquer código do épico, conforme exige a Regra Zero (`docs/AGENTS.md`) e a dependência declarada no backlog ("ADR-001 aprovado antes de iniciar qualquer história").

### Restrições de arquitetura aplicáveis

- **ADR-004:** `io.spring.graphql.*` é código gerado — nunca editar manualmente; excluído do Pitest.
- **ADR-001:** manter DGS 10.x — sem reescrita de resolvers.

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Garante que o **ADR-001** em `docs/06-architecture-decisions.md` esteja completo: contexto, opções (A e B), decisão (Opção A — DGS 10.x), consequências e status `✅ Aceito`
- Registra a rastreabilidade do ADR para INI-04 e para as histórias dependentes (US-04.03, US-04.06)
- Confirma que a decisão não viola ADR-004 (código gerado intocado)

### ❌ O que esta história NÃO faz

- Não altera `build.gradle` nem versões — isso é US-04.06 (DGS) e US-04.02/04.03 (Java/Gradle/Spring Boot)
- Não toca em nenhum resolver GraphQL
- Não edita `io.spring.graphql.*`
