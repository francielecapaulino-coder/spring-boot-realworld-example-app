## Contexto

Esta história é a **validação consolidada** do EPIC-04: não introduz mudança funcional, apenas verifica que o conjunto Java 25 + Spring Boot 4.0.3 + Gradle 9.3.1 + Jakarta + Security 6.x + DGS 10.x + `java.time` + virtual threads funciona sem regressões, atendendo o **Marco M3**.

A rede de segurança de testes (EPIC-07 mutação, EPIC-08 contrato) deve estar verde. O threshold de Pitest (≥ 95%, ADR/AGENTS.md) **não pode ser reduzido**.

### Restrições de arquitetura aplicáveis

- **Pitest ≥ 95%** — nunca reduzir (AGENTS.md / lista vermelha).
- **ADR-004:** `io.spring.graphql.*` excluído do Pitest.
- Nenhuma mudança de contrato de API.

---

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Executa e documenta a suíte completa: `./gradlew build`, `test`, `pitest`
- Verifica o checklist do Marco M3 (Java 25, Gradle 9.3.1 sem warnings, 0 `javax.*` Jakarta, 0 Joda-Time, virtual threads, GraphQL DGS 10.x)
- Consolida a evidência de conclusão do épico
- Reporta e (se houver) encaminha regressões residuais

### ❌ O que esta história NÃO faz

- Não introduz mudança funcional ou de configuração
- Não migra MyBatis → JPA (EPIC-05) nem introduz records (EPIC-06)

## Verificações obrigatórias

```bash
# V1 — build sem deprecation
./gradlew build --warning-mode all 2>&1 | grep -i "deprecated" || echo "0 deprecation warnings"

# V2 — testes
JWT_SECRET=$(openssl rand -base64 64) ./gradlew test --console=plain

# V3 — mutação
JWT_SECRET=$(openssl rand -base64 64) ./gradlew pitest --console=plain
# Esperado: mutation score >= 95%

# V4 — runtime
./gradlew --version            # Gradle 9.3.1
docker compose run --rm app java -version   # Java 25 (no container)

# V5 — jakarta / joda
grep -rn "import javax\.\(persistence\|validation\|servlet\|annotation\)" src/ || echo "0 javax Jakarta"
grep -i "joda" build.gradle || echo "0 joda"

# V6 — GraphQL (app no ar)
curl -s -X POST http://localhost:8080/graphql -H "Content-Type: application/json" -d '{"query":"{ tags }"}'
```
