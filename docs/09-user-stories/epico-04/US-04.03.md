## História de usuário

**Como** engenheiro responsável pela modernização do runtime,
**quero** que o framework seja atualizado para Spring Boot 4.0.3, com todas as dependências resolvidas e o build verde —
**para que** o projeto rode sobre uma linha de framework com suporte ativo e habilite Jakarta EE 11, Spring Security 6.x e DGS 10.x.

---

## Contexto

O salto de **Spring Boot 2.6.3** (EOL) para **4.0.3** atravessa a major 3.x e implica:

- **Jakarta EE 11:** todos os `javax.*` passam a `jakarta.*` (tratado em US-04.04, mas o build não compila até lá — esta história pode quebrar a compilação temporariamente, encadeando com US-04.04/04.05/04.06 na mesma sequência de épico).
- **Spring Security 6.x:** `WebSecurityConfigurerAdapter` removido (US-04.05).
- **DGS:** versão 10.x compatível com Spring Boot 4 (US-04.06).
- **Atualizações transitivas:** `spring-boot-starter-*`, Testcontainers, RestAssured, MyBatis starter (até EPIC-05), micrometer/actuator, flyway.

Por ser de alto risco e afetar todo o codebase, EPIC-07 (mutação) e EPIC-08 (contrato) atuam como rede de segurança em paralelo (dependência declarada no backlog).

### Restrições de arquitetura aplicáveis

- **ADR-001:** GraphQL permanece em DGS (10.x) — sem reescrita de resolvers.
- **ADR-004:** `io.spring.graphql.*` é gerado — intocado.
- **ADR-006:** `JWT_SECRET` sem fallback; `JWT_SESSION_TIME` com fallback 86400 — preservar.
- Nenhuma mudança de contrato de API (`docs/01.1-api-mapping.md` v2.0).

---

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Atualiza a versão do plugin/BOM Spring Boot para **4.0.3** no `build.gradle`
- Resolve conflitos de dependências e versões transitivas para compatibilidade com 4.0.3
- Ajusta propriedades de configuração renomeadas/removidas entre 2.6 e 4.0
- Coordena com US-04.04/04.05/04.06 para que o build volte a compilar e os testes passem ao final do encadeamento

### ❌ O que esta história NÃO faz

- Não realiza a migração `javax.*` → `jakarta.*` linha a linha — isso é US-04.04
- Não reconfigura o Spring Security — isso é US-04.05
- Não atualiza o DGS — isso é US-04.06
- Não migra MyBatis → JPA (EPIC-05) nem introduz records (EPIC-06)

## Verificações obrigatórias

```bash
# V1 — versão Spring Boot
grep -n "id 'org.springframework.boot'" build.gradle
grep -n "4.0.3" build.gradle

# V2 — resolução de dependências
./gradlew dependencies --configuration runtimeClasspath > /tmp/deps.txt 2>&1; tail -5 /tmp/deps.txt

# V3 — build (ao final do encadeamento US-04.03..04.06)
JWT_SECRET=$(openssl rand -base64 64) ./gradlew build --console=plain

# V4 — testes
JWT_SECRET=$(openssl rand -base64 64) ./gradlew test --console=plain
# Esperado: BUILD SUCCESSFUL
```
