## História de usuário

**Como** engenheiro responsável pela modernização do runtime,
**quero** que todos os imports `javax.*` sejam migrados para `jakarta.*` —
**para que** o código compile e execute sobre o Jakarta EE 11 exigido pelo Spring Boot 4.0.3.

---

## Contexto

O Spring Boot 4 usa **Jakarta EE 11**: o namespace `javax.*` foi substituído por `jakarta.*`. As ocorrências esperadas no codebase incluem:

- `javax.persistence.*` (entidades/anotações — onde aplicável)
- `javax.validation.*` (Bean Validation nos DTOs)
- `javax.servlet.*` (filtros, ex.: `JwtTokenFilter`)
- `javax.crypto.*` — **`DefaultJwtService.java`** usa `javax.crypto.SecretKey` e `javax.crypto.spec.SecretKeySpec`

> **Atenção:** `javax.crypto.*` faz parte do **JDK** (não do Jakarta EE) e **permanece `javax.crypto`** — não deve ser renomeado para `jakarta.crypto`. A regra "0 imports `javax.*`" do épico refere-se ao **namespace Jakarta EE** (`persistence`, `validation`, `servlet`, `annotation`, etc.). Esta história documenta explicitamente essa exceção para evitar quebra do `DefaultJwtService`.

### Restrições de arquitetura aplicáveis (lista vermelha / RATCHET)

- **Nunca usar `import javax.*`** em código novo (AGENTS.md) — exceto `javax.crypto.*` do JDK, conforme nota acima.
- **ADR-004:** não editar `io.spring.graphql.*`.
- **ADR-006:** `DefaultJwtService` deve preservar o comportamento de assinatura/validação JWT.

---

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Migra todos os imports Jakarta EE `javax.{persistence,validation,servlet,annotation,...}` → `jakarta.*`
- Ajusta o `DefaultJwtService` mantendo `javax.crypto.*` (JDK) e migrando eventuais `javax.*` Jakarta presentes
- Garante 0 imports `javax.*` de namespace Jakarta no `src/`
- Mantém o build compilando após a migração

### ❌ O que esta história NÃO faz

- Não reconfigura o Spring Security — isso é US-04.05
- Não atualiza o DGS — isso é US-04.06
- Não remove Joda-Time — isso é US-04.07
- Não altera contrato de API nem comportamento de negócio

## Verificações obrigatórias

```bash
# V1 — sem javax Jakarta EE
grep -rn "import javax\.\(persistence\|validation\|servlet\|annotation\)" src/ || echo "0 imports javax (Jakarta EE)"

# V2 — javax.crypto preservado (JDK)
grep -rn "javax.crypto" src/main/java || echo "nenhum (ok se DefaultJwtService usar outra API)"

# V3 — compila
```
