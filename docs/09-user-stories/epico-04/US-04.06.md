## História de usuário

**Como** engenheiro responsável pela modernização do runtime,
**quero** que o Netflix DGS Framework seja atualizado para a versão 10.x compatível com Spring Boot 4 —
**para que** a API GraphQL continue operacional sem reescrever nenhum resolver (conforme ADR-001).

---

## Contexto

Conforme **ADR-001** (US-04.01), a estratégia é **manter o DGS** e atualizá-lo para a **versão 10.x**, que é internamente reimplementada sobre o Spring for GraphQL e é compatível com Spring Boot 4. As anotações `@DgsComponent`, `@DgsQuery`, `@DgsMutation` permanecem inalteradas — **apenas a versão muda no `build.gradle`**.

O pacote `io.spring.graphql.*` é **código gerado** pela tarefa `generateJava` do DGS codegen — **nunca editar manualmente** (ADR-004). Após o upgrade, `./gradlew generateJava` deve regenerar o pacote sem erros e `POST /graphql` deve responder.

### Restrições de arquitetura aplicáveis

- **ADR-001:** sem reescrita de resolvers — apenas atualização de versão.
- **ADR-004:** `io.spring.graphql.*` é gerado — intocado; excluído do Pitest.
- **ADR-005:** `ArticleData` implementa `Node` com `getCursor()` explícito — preservar (Joda-Time é tratado na US-04.07).

---

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Atualiza a versão do DGS Framework e do DGS codegen plugin para **10.x** no `build.gradle`
- Garante que `./gradlew generateJava` regenere `io.spring.graphql.*` sem erro
- Confirma que `POST /graphql` com `{ "query": "{ tags }" }` retorna resultado válido
- Mantém todos os resolvers existentes sem alteração de anotações

### ❌ O que esta história NÃO faz

- Não reescreve resolvers (`@DgsQuery` permanece — ADR-001)
- Não edita `io.spring.graphql.*` manualmente (ADR-004)
- Não remove Joda-Time — isso é US-04.07
- Não altera o schema GraphQL (`schema.graphqls`)

## Verificações obrigatórias

```bash
# V1 — versão DGS
grep -ni "dgs" build.gradle

# V2 — codegen regenera
JWT_SECRET=$(openssl rand -base64 64) ./gradlew generateJava --console=plain

# V3 — GraphQL operacional (com app no ar)
curl -s -X POST http://localhost:8080/graphql -H "Content-Type: application/json" -d '{"query":"{ tags }"}'
# Esperado: JSON válido com tags

# V4 — testes
JWT_SECRET=$(openssl rand -base64 64) ./gradlew test --console=plain
# Esperado: BUILD SUCCESSFUL
```
