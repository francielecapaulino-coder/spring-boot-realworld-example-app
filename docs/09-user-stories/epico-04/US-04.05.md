## História de usuário

**Como** engenheiro responsável pela modernização do runtime,
**quero** que a configuração de segurança use o modelo do Spring Security 6.x (`SecurityFilterChain` via `@Bean`, sem herança de `WebSecurityConfigurerAdapter`) —
**para que** a aplicação suba no Spring Boot 4.0.3 preservando exatamente as mesmas regras de autorização atuais.

---

## Contexto

A classe `io.spring.api.security.WebSecurityConfig` hoje **estende `WebSecurityConfigurerAdapter`** e sobrescreve `configure(HttpSecurity http)`. No Spring Security 6.x (Spring Boot 4) esse adapter foi removido. O novo modelo:

```
ANTES: class WebSecurityConfig extends WebSecurityConfigurerAdapter
       → @Override configure(HttpSecurity http)

DEPOIS: class WebSecurityConfig  (sem herança)
        → @Bean SecurityFilterChain filterChain(HttpSecurity http)
```

As **regras de autorização atuais devem ser preservadas integralmente**, incluindo:
- `permitAll` para `OPTIONS`, `/graphiql`, `/graphql`, `POST /users` e `/users/login`, `GET /articles/**`/`/profiles/**`/`/tags`;
- **`permitAll` GET para `/actuator/{health,info,prometheus,metrics}`** (adicionado na US-03.07 / issue #67 — não pode regredir);
- `authenticated()` para `GET /articles/feed` e `anyRequest()`;
- `JwtTokenFilter` antes de `UsernamePasswordAuthenticationFilter`;
- CORS e `SessionCreationPolicy.STATELESS` inalterados.

### Restrições de arquitetura aplicáveis

- Módulo de autenticação (`io.spring.api.security.*`) — alteração sensível; preservar contrato de segurança.
- **ADR-006:** comportamento de JWT inalterado.
- Regressão coberta por `ActuatorSecurityTest` (US-03.07) e testes de API (EPIC-08).

---

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Remove a herança de `WebSecurityConfigurerAdapter` em `WebSecurityConfig`
- Expõe `@Bean SecurityFilterChain filterChain(HttpSecurity http)` com as **mesmas regras** de autorização
- Mantém `JwtTokenFilter`, CORS e política STATELESS
- Garante que `ActuatorSecurityTest` (US-03.07) continue verde (regras de `/actuator/*` preservadas)

### ❌ O que esta história NÃO faz

- Não altera as regras de autorização (apenas o modelo de configuração)
- Não mexe no algoritmo/segredo JWT (ADR-006)
- Não atualiza o DGS (US-04.06) nem remove Joda-Time (US-04.07)
