## Escopo

* Remover todos os arquivos Java listados acima
* Remover todos os XMLs de `src/main/resources/mapper/`
* Remover `mybatis-spring-boot-starter` do `build.gradle`
* Remover configuração `mybatis.*` do `application.properties`
* Remover `logging.level` de pacotes MyBatis do `application.properties`
* Verificar que nenhuma classe remanescente importa `org.apache.ibatis.*`
* `./gradlew build` deve compilar sem erros após cada remoção

## Fora de escopo

* Remoção do diretório `src/main/resources/mapper/` vazio — remover apenas o conteúdo; se o diretório ficar vazio, removê-lo também
* Mudança no schema do banco (`V1__create_tables.sql`) — inalterado
* Mudança nas interfaces do `core` — inalteradas

## Critérios de aceite

* [ ] `grep -r "mybatis" build.gradle` → 0 resultados
* [ ] `grep -r "mybatis" application.properties` → 0 resultados
* [ ] `find src -name "*Mapper.java"` → 0 resultados
* [ ] `find src -name "*Mapper.xml"` → 0 resultados
* [ ] `find src -name "MyBatis*.java"` → 0 resultados
* [ ] `grep -r "import org.apache.ibatis" src/` → 0 resultados
* [ ] `./gradlew build` sem erros
* [ ] `./gradlew test` 100% verde

## Regras técnicas

* Remover em blocos lógicos: primeiro os XMLs, depois os mappers Java, depois os repositórios, por último a dependência do `build.gradle`
* Após remover a dependência do `build.gradle`, fazer `./gradlew build` imediatamente para confirmar que nada está quebrado
