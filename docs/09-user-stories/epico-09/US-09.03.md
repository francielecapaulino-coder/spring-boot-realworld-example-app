## Contexto

Os 3 fluxos MT são os fluxos mínimos que cobrem o core da aplicação sem depender de EPIC-12 ou EPIC-13.

**Fluxos MT (3):**

| Fluxo | Sequência | Validação |
|---|---|---|
| **F1 — Registro e Login** | `POST /users` → `POST /users/login` → `GET /user` | Token JWT retornado; perfil carregado |
| **F2 — Criar e ler artigo** | Login → `POST /articles` → `GET /articles/:slug` | Campos do artigo corretos; autor correto |
| **F3 — Interação social** | Login como A → `POST /profiles/B/follow` → `GET /articles/feed` | Artigos de B aparecem no feed de A |

## Escopo

* No branch `feat/playwright-working`, implementar os 3 fluxos MT com assertions funcionando
* Usar Playwright como cliente HTTP (não browser) — `request` API
* Cada fluxo deve ser um arquivo de teste independente
* CI verde no branch `feat/playwright-working`
* Documentar evidência em `docs/process/tests-epico-09/working-branch-mt-evidence.md`

## Fora de escopo

* Limpar dados criados no `afterEach` quando possível
