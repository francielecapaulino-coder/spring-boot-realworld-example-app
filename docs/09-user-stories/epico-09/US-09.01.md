## História de usuário

**Como** QA automatizado responsável pela validação E2E,  
**quero** configurar Playwright e escrever testes que chamam a interface web da aplicação (se houver) ou falam com a API via HTTP em modo navegador,  
**para que** haja validação realista de ponta a ponta antes do deployment.

---

## Contexto

O projeto é backend-only (REST e GraphQL). Não há UI web própria. Configuraremos Playwright Node.js para simular client web que faz chamadas HTTP diretamente aos endpoints REST, para testes E2E com cookies e token JWT, incluindo navegação de usuário 1 (signup/login) e autor 2 (criar artigo, favoritar). Ou eventualmente future UI web; por ora, a API será o target. também incluir setup do Playwright com Docker se possível.

---

## Escopo

* Inicializar Playwright no repo: `npx playwright init`
* Configurar `playwright.config.ts` com baseURL `http://localhost:8080/api`
* Escrever `e2e/auth.spec.ts` cobrindo signup e login via API
* Escrever `e2e/article.spec.ts` cobrindo CRUD de artigo com JWT token
* Escrever `e2e/profile.spec.ts` pegando follow/unfollow
* Add `@playwright/test` ao `package.json` e scripts de test
* Incluir script `npm run test:e2e` localizable por CI
* Usar Docker Compose para levantar app e DB nos testes (exemplo com `test:servers`)

## Fora de escopo

* Testes com seletor de UI (ainda não há UI)
* Parallelização de testes regional
* Reportagem com pacto e screenshots

## Critérios de aceite

* [ ] `npx playwright install` installs browsers OK na máquina
* [ ] `npm run test:e2e` roda os 3 specs e passam com app em Docker
* [ ] Especificações validam 200, 401, 403 e criam entidades com sucesso
* [ ] Playwright config usa baseURL e pode rodar headless

## Regras técnicas

* Usar `request` do Playwright para chamadas HTTP diretas; página não utilizada agora
* Usar variáveis de ambiente para baseURL e credenciais
* Fixar baseURProxy se CI bloqueia localhost

## Dependências

* EPIC-03 (Docker+Compose do serviço)
* EPIC-04 (runtime moderno, endpoints respondendo)

## Resultado esperado

Playwright configurado com testes E2E que validam fluxos principais da API usando request em modo navegador, prontos para isolamento em branch broken e working.
