## Contexto

Após US-06.01 a US-06.05, a maioria dos DTOs terá sido convertida. Esta história serve como gate de qualidade: conta as classes `record` vs classes DTO/value object ainda como classes Lombok, calcula o percentual e verifica se o threshold de 80% foi atingido.

**Critério de elegibilidade (do INI-06):** uma classe conta como DTO para o threshold se:
- É DTO, objeto de transferência ou value object (não é `@Entity`, `@Service`, `@Repository`, controller, exception, etc.)
- Está nos pacotes `io.spring.application.data`, `io.spring.application.user`, `io.spring.application.article`, `io.spring.api.exception` (para recursos), `io.spring.core` (value objects não-entidade)

**Classes inelegíveis (não entram no denominador):**
- `Page`, `CursorPageParameter` — lógica de validação em setters, estado mutável necessário
- `UpdateUserParam` — `@Builder.Default` implica mutabilidade
- `DateTimeCursor`, `PageCursor` — herança de classe abstrata
- `UpdateArticleParam` — se mantida como classe (ver US-06.03)
- `Tag`, `FollowRelation` — se tiverem `@Entity` após EPIC-05
- Entidades JPA (`Article`, `Comment`, `User`, etc.) — nunca elegíveis

---

## Escopo

* **Contar** as classes DTO/value object elegíveis no codebase após US-06.01–06.05
* **Calcular** percentual: `(número de records) / (número de classes elegíveis) × 100`
* **Verificar** se resultado ≥ 80%
* **Se < 80%:** identificar quais classes ainda são Lombok e que são elegíveis, converter as restantes até atingir 80%
* **Documentar** resultado final em `docs/process/tests-epico-06/` com tabela de todas as classes, status (record / classe / inelegível) e percentual final
* **Verificar Lombok residual** nos pacotes da aplicação: remover imports `lombok.*` de qualquer classe já convertida onde o import ficou esquecido
* Garantir que `./gradlew build` passa limpo (sem warnings) e `./gradlew test` 100% verde

## Fora de escopo

* Conversão de entidades JPA — proibido pelo critério de elegibilidade
* Conversão de classes com estado mutável necessário (ex: `Page`)
* Remoção completa do Lombok do projeto — Lombok permanece em classes inelegíveis; esta história apenas remove Lombok das classes já convertidas que ainda têm imports esquecidos
