## História de usuário

**Como** PM que precisa validar que o EPIC-03 cumpriu o critério de aceite de onboarding,
**quero** que um desenvolvedor real execute o guia do `CONTRIBUTING.md` em um ambiente limpo e que o tempo total seja medido e documentado —
**para que** possamos confirmar com evidência que o objetivo "qualquer pessoa clona e tem o sistema rodando em menos de 15 minutos" foi alcançado.

---

## Contexto

O critério de aceite do EPIC-03 exige: **"Dev novo subiu o ambiente em ≤ 15 minutos — tempo documentado"**.

Esta história é a prova de que todos os artefatos dos US-03.01 a US-03.08 funcionam em conjunto. A medição é feita em ambiente limpo (sem cache Docker, sem dependências pré-instaladas além dos pré-requisitos listados no `CONTRIBUTING.md`).

### O que é "ambiente limpo"

| Item | Condição |
|---|---|
| Docker cache | Limpo (`docker system prune -af`) |
| Repositório | Clonado do zero (não de uma cópia local existente) |
| `.env` | Criado durante a medição a partir do `.env.example` |
| Pré-requisitos | Apenas os listados no `CONTRIBUTING.md` (Docker, Python 3, Git) |

### Definição de "tempo de onboarding"

Começa em: `git clone <url>`
Termina em: `curl http://localhost:8080/actuator/health` → `{"status":"UP"}`

---

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Executa o onboarding seguindo o `CONTRIBUTING.md` em ambiente limpo
- Cronometra cada passo (clone, `.env`, `docker compose up`, verificação)
- Documenta o resultado em `docs/process/tests-epico-03/test-evidence-us-03.09.md`
- Reporta problemas encontrados durante a medição e os resolve (se houver)
- Confirma o critério "≤ 15 minutos" com evidência

### ❌ O que esta história NÃO faz

- Não altera código de produção — apenas documenta
- Não faz medição em staging ou produção — ambiente local apenas
- Não requer um desenvolvedor externo — a PM ou qualquer membro do time pode fazer a medição

## Verificações ao final do onboarding

```bash
# V1 — Saúde da aplicação
curl -s http://localhost:8080/actuator/health
# Esperado: {"status":"UP",...}

# V2 — Endpoint de negócio
curl -s http://localhost:8080/tags
# Esperado: {"tags":[...]}

# V3 — Grafana acessível
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000
# Esperado: 200 ou 302

# V4 — Todos os serviços saudáveis
docker compose ps
# Esperado: 6 serviços rodando

# V5 — Script de validação
python3 scripts/validate_startup.py
# Esperado: exit code 0
```
