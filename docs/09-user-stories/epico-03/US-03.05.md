## História de usuário

**Como** desenvolvedor que acessa o Grafana pela primeira vez após `docker compose up`,
**quero** que as fontes de dados Prometheus, Loki e Tempo já estejam configuradas automaticamente —
**para que** não seja necessária nenhuma configuração manual no Grafana para começar a explorar métricas, logs e traces.

---

## Contexto

A regra do épico define: **Grafana pré-configurado via provisioning** — fontes de dados não dependem de configuração manual.

O Grafana suporta provisioning automático via arquivos YAML montados como volume em `/etc/grafana/provisioning/`. Ao iniciar, o Grafana lê esses arquivos e configura as fontes de dados sem intervenção humana.

### Fontes de dados a provisionar

| Fonte de dados | URL interna (Docker network) | Tipo |
|---|---|---|
| Prometheus | `http://prometheus:9090` | `prometheus` |
| Loki | `http://loki:3100` | `loki` |
| Tempo | `http://tempo:3200` | `tempo` |

### Estrutura de arquivos

```
grafana/
└── provisioning/
    └── datasources/
        └── datasources.yml   ← define as 3 fontes de dados
```

O `docker-compose.yml` (US-03.04) monta este diretório como volume:
```yaml
grafana:
  volumes:
    - ./grafana/provisioning:/etc/grafana/provisioning
```

---

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Cria o diretório `grafana/provisioning/datasources/`
- Cria `grafana/provisioning/datasources/datasources.yml` com as 3 fontes de dados
- Atualiza o serviço `grafana` no `docker-compose.yml` para montar o volume de provisioning
- Verifica que após `docker compose up`, as fontes de dados aparecem em `http://localhost:3000/datasources`

### ❌ O que esta história NÃO faz

- Não cria dashboards Grafana — isso é EPIC-10
- Não configura alertas — isso é EPIC-10
- Não configura autenticação Grafana além do padrão `admin/admin` — fora do escopo
- Não configura Prometheus scrape targets além do `/actuator/prometheus` — isso é EPIC-10

## Verificações obrigatórias

```bash
# V1 — Arquivos de provisioning
ls grafana/provisioning/datasources/datasources.yml
# Esperado: arquivo presente

# V2 — Conteúdo das fontes de dados
grep -E "prometheus:9090|loki:3100|tempo:3200" grafana/provisioning/datasources/datasources.yml
# Esperado: 3 linhas

# V3 — Volume no compose
grep "provisioning" docker-compose.yml
# Esperado: ./grafana/provisioning:/etc/grafana/provisioning

# V4 — Subir stack e verificar Grafana
docker compose up -d
sleep 30
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000
# Esperado: 200 ou 302

# V5 — Verificar fontes de dados via API Grafana
curl -s -u admin:admin http://localhost:3000/api/datasources | python3 -c "import json,sys; ds=json.load(sys.stdin); print(len(ds), 'datasources')"
# Esperado: 3 datasources

# V6 — Cleanup
docker compose down
```

---

## Conteúdo do `datasources.yml`

```yaml
apiVersion: 1

datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
    editable: false

  - name: Loki
    type: loki
    access: proxy
    url: http://loki:3100
    editable: false

  - name: Tempo
    type: tempo
    access: proxy
    url: http://tempo:3200
    editable: false
```
