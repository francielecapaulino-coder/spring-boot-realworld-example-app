## História de usuário

**Como** PM que quer garantir que nenhum PR quebre o ambiente Docker Compose,
**quero** que o script `scripts/validate_startup.py` seja executado automaticamente no CI a cada pull request —
**para que** qualquer regressão no `Dockerfile`, no `docker-compose.yml` ou nas migrations seja detectada antes do merge.

---

## Contexto

O critério de aceite do EPIC-03 exige: **CI executa o script e falha se exit code ≠ 0**.

O workflow atual (`.github/workflows/gradle.yml`) já tem os jobs `secret-scan` e `build`. Esta história adiciona um terceiro job `validate-startup` que:

1. Faz checkout do código
2. Instala Python e a dependência `requests`
3. Cria o arquivo `.env` com `JWT_SECRET` gerado dinamicamente e `POSTGRES_PASSWORD` gerado dinamicamente
4. Executa `python3 scripts/validate_startup.py`
5. Falha o CI se exit code ≠ 0

### Ordem de jobs no CI

```
secret-scan → build → validate-startup
```

O job `validate-startup` **não depende** de `build` (são independentes — validam coisas diferentes), mas deve rodar **após** `secret-scan` por segurança.

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Adiciona job `validate-startup` ao `.github/workflows/gradle.yml`
- Job usa `ubuntu-latest` com Docker e Docker Compose disponíveis
- Gera `JWT_SECRET` e `POSTGRES_PASSWORD` dinamicamente via `openssl rand`
- Cria `.env` temporário para o Docker Compose
- Executa `python3 scripts/validate_startup.py` e verifica exit code
- Executa `docker compose down` no `post` step para limpeza

### ❌ O que esta história NÃO faz

- Não roda testes E2E no CI — isso é EPIC-09
- Não faz deploy — fora do escopo
- Não gera relatórios de performance — isso é EPIC-10

## Estrutura do job no workflow

```yaml
validate-startup:
  name: Validate Docker Compose startup
  runs-on: ubuntu-latest
  needs: secret-scan

  steps:
    - uses: actions/checkout@v4

    - name: Set up Python 3
      uses: actions/setup-python@v5
      with:
        python-version: '3.x'

    - name: Install script dependencies
      run: pip install requests

    - name: Generate secrets for Docker Compose
      run: |
        echo "JWT_SECRET=$(openssl rand -base64 64 | tr -d '\n')" >> $GITHUB_ENV
        echo "POSTGRES_PASSWORD=$(openssl rand -base64 32 | tr -d '\n')" >> $GITHUB_ENV

    - name: Create .env file for Docker Compose
      run: |
        echo "JWT_SECRET=${JWT_SECRET}" > .env
        echo "JWT_SESSION_TIME=604800" >> .env
        echo "POSTGRES_PASSWORD=${POSTGRES_PASSWORD}" >> .env
        echo "POSTGRES_DB=realworld" >> .env
        echo "POSTGRES_USER=postgres" >> .env
        echo "SPRING_PROFILES_ACTIVE=dev" >> .env

    - name: Validate Docker Compose startup and shutdown
      run: python3 scripts/validate_startup.py

    - name: Cleanup Docker Compose
      if: always()
      run: docker compose down --volumes --remove-orphans
```
