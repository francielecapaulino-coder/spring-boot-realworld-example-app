## História de usuário

**Como** desenvolvedor que quer rodar a aplicação em qualquer máquina sem configurar JDK localmente,
**quero** que exista um `Dockerfile` multi-stage que compile e empacote a aplicação Spring Boot em uma imagem Docker enxuta —
**para que** qualquer membro do time possa construir e executar a aplicação com `docker build` e `docker run`, sem dependência de ambiente local Java.

---

## Contexto

O projeto atualmente não possui `Dockerfile`. Para rodar a aplicação, é necessário ter JDK 11 instalado localmente e executar `./gradlew bootRun`. Isso cria fricção no onboarding e impede que o Docker Compose (US-03.04) orquestre a aplicação como serviço junto ao PostgreSQL e à stack LGTM.

### Estratégia multi-stage

| Stage | Imagem base | Propósito |
|---|---|---|
| `build` | `eclipse-temurin:11-jdk-alpine` | Compila e gera o `.jar` via Gradle |
| `runtime` | `eclipse-temurin:11-jre-alpine` | Executa o `.jar` — imagem enxuta sem JDK |

### Variáveis de ambiente obrigatórias (injetadas pelo Docker Compose)

| Variável | Obrigatoriedade | Default |
|---|---|---|
| `JWT_SECRET` | **Obrigatória** — sem ela o container não sobe | Nenhum (fail-fast via US-02.02) |
| `JWT_SESSION_TIME` | Opcional | `604800` (7d — perfil dev via US-02.04) |
| `SPRING_PROFILES_ACTIVE` | Opcional | `dev` |

---

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Cria `Dockerfile` multi-stage na raiz do repositório
- Stage `build`: copia o projeto, executa `./gradlew bootJar`, gera `app.jar`
- Stage `runtime`: copia apenas o `.jar` gerado, expõe porta `8080`
- Usa `ENTRYPOINT` com `java -jar` e suporte a variáveis de ambiente via `-e`
- Adiciona `.dockerignore` para excluir `build/`, `.git/`, `*.md` e reduzir contexto de build

### ❌ O que esta história NÃO faz

- Não cria `docker-compose.yml` — isso é US-03.04
- Não configura Spring Actuator — isso é US-03.02
- Não substitui SQLite por PostgreSQL — isso é US-03.03
- Não faz push para registry — ambiente local apenas

| CA | Tipo | Descrição | Verificação |
|---|---|---|---|
| CA-01 | Estrutural | `Dockerfile` presente na raiz do repositório | `ls Dockerfile` |
| CA-02 | Estrutural | `.dockerignore` presente na raiz do repositório | `ls .dockerignore` |
| CA-03 | Build | Stage `build` usa `eclipse-temurin:11-jdk-alpine` | `grep "FROM.*jdk" Dockerfile` |

| CA-05 | Build | `docker build -t realworld-app .` → exit 0, imagem gerada | `docker images realworld-app` |
| CA-06 | Execução | `docker run -e JWT_SECRET=<valor> -p 8080:8080 realworld-app` → app sobe | log `Started RealworldApplication` |
| CA-07 | Execução | `curl http://localhost:8080/tags` → HTTP 200 com JSON | resposta com `{"tags":[...]}` |
| CA-08 | Segurança | `docker run` sem `JWT_SECRET` → container termina com mensagem clara | log `JWT_SECRET nao esta definida` |
| CA-09 | Tamanho | Imagem `runtime` menor que 300 MB | `docker image inspect realworld-app --format='{{.Size}}'` |

## Verificações obrigatórias

```bash
# V1 — Build da imagem
docker build -t realworld-app .
# Esperado: Successfully built <hash>

# V2 — Execução com JWT_SECRET
docker run -d --name realworld-test \
  -e JWT_SECRET=$(openssl rand -base64 64) \
  -e SPRING_PROFILES_ACTIVE=dev \
  -p 8080:8080 realworld-app

# V3 — Health check
sleep 10 && curl -s http://localhost:8080/tags
# Esperado: {"tags":[...]}

# V4 — Fail-fast sem JWT_SECRET
docker run --rm realworld-app
# Esperado: container termina com mensagem sobre JWT_SECRET

# V5 — Tamanho da imagem
docker image inspect realworld-app --format='{{.Size}}'
# Esperado: < 314572800 (300 MB)

# V6 — Cleanup
docker stop realworld-test && docker rm realworld-test
```

== ETAPA 1/3 — CRIAR DOCKERFILE ==
Arquivo: Dockerfile (raiz do repositório)

Stage build:
  FROM eclipse-temurin:11-jdk-alpine AS build
  WORKDIR /app
  COPY . .
  RUN ./gradlew bootJar --no-daemon

Stage runtime:
  FROM eclipse-temurin:11-jre-alpine
  WORKDIR /app
  COPY --from=build /app/build/libs/*.jar app.jar
  EXPOSE 8080
  ENTRYPOINT ["java", "-jar", "app.jar"]

== ETAPA 2/3 — CRIAR .dockerignore ==
Arquivo: .dockerignore (raiz do repositório)
Conteúdo mínimo:
  build/
