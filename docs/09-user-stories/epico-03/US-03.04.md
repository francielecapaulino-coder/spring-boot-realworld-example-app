## História de usuário

**Como** desenvolvedor que quer subir toda a stack de desenvolvimento com um único comando,
**quero** que exista um `docker-compose.yml` que orquestre a aplicação Spring Boot, PostgreSQL 16 e a stack LGTM (Prometheus, Loki, Tempo e Grafana) —
**para que** qualquer pessoa possa executar `docker compose up` e ter o sistema completo rodando em menos de 15 minutos, sem configuração manual.

---

## Contexto

### Arquitetura do ambiente Docker Compose

```
docker-compose.yml
│
├── app (porta 8080)
│   ├── Depende de: postgres (healthcheck via /actuator/health)
│   └── Variáveis: JWT_SECRET, JWT_SESSION_TIME, SPRING_PROFILES_ACTIVE=dev
│       POSTGRES_HOST=postgres, POSTGRES_PORT=5432, POSTGRES_DB=realworld
│
├── postgres (porta 5432) — postgres:16-alpine
│   └── Variáveis: POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD
│
├── prometheus (porta 9090) — coleta de /actuator/prometheus
├── loki (porta 3100) — logs estruturados
├── tempo (porta 3200) — traces
└── grafana (porta 3000) — dashboards (admin/admin)
    └── Depende de: prometheus, loki, tempo
```

### Regras do épico

- **`docker compose up` → 6 serviços `healthy`** — critério de aceite obrigatório
- **`JWT_SECRET` injetado pelo compose** — lido do arquivo `.env` na raiz (US-02.03)
- **Grafana pré-configurado via provisioning** — fontes de dados sem configuração manual (US-03.05)

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Cria `docker-compose.yml` na raiz do repositório com os 6 serviços
- Configura `healthcheck` para cada serviço
- Injeta `JWT_SECRET` e `JWT_SESSION_TIME` no serviço `app` via arquivo `.env`
- Configura volumes para PostgreSQL (dados persistentes) e Grafana (dashboards)
- Configura redes internas entre os serviços
- Atualiza `.env.example` com variáveis PostgreSQL necessárias

### ❌ O que esta história NÃO faz

- Não configura fontes de dados do Grafana — isso é US-03.05
- Não cria dashboards Grafana — isso é EPIC-10
- Não valida startup programaticamente — isso é US-03.06
- Não documenta o onboarding no CONTRIBUTING.md — isso é US-03.08

## Critérios de aceite

| CA | Tipo | Descrição | Verificação |
|---|---|---|---|
| CA-01 | Estrutural | `docker-compose.yml` presente na raiz | `ls docker-compose.yml` |
| CA-02 | Serviços | 6 serviços definidos: `app`, `postgres`, `prometheus`, `loki`, `tempo`, `grafana` | `docker compose config --services` |
| CA-03 | Healthcheck | `app` usa `/actuator/health` no healthcheck | `grep "actuator/health" docker-compose.yml` |
| CA-04 | Dependência | `app` tem `depends_on: postgres: condition: service_healthy` | verificação no YAML |
| CA-05 | Variáveis | `JWT_SECRET` e `JWT_SESSION_TIME` injetados no serviço `app` | `grep "JWT_SECRET" docker-compose.yml` |
| CA-06 | Execução | `docker compose up -d` → todos os 6 serviços em estado `healthy` | `docker compose ps` |
| CA-07 | Funcional | `curl http://localhost:8080/tags` → HTTP 200 com JSON | `{"tags":[...]}` |
| CA-08 | Funcional | `curl http://localhost:8080/actuator/health` → `{"status":"UP"}` | body contém `"UP"` |
| CA-09 | Funcional | Grafana acessível em `http://localhost:3000` | HTTP 200 ou 302 |
| CA-10 | Cleanup | `docker compose down` → todos os serviços parados | `docker compose ps` → 0 running |

## Configuração dos serviços

### Serviço `app`

```yaml
app:
  build: .
  ports: ["8080:8080"]
  environment:
    JWT_SECRET: ${JWT_SECRET}
    JWT_SESSION_TIME: ${JWT_SESSION_TIME:-604800}
    SPRING_PROFILES_ACTIVE: dev
    POSTGRES_HOST: postgres
    POSTGRES_PORT: 5432
    POSTGRES_DB: ${POSTGRES_DB:-realworld}
    POSTGRES_USER: ${POSTGRES_USER:-postgres}
    POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
  depends_on:
    postgres:
      condition: service_healthy
  healthcheck:
    test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
    interval: 30s
    timeout: 10s
    retries: 5
    start_period: 60s
```

### Serviço `postgres`

```yaml
postgres:
  image: postgres:16-alpine
  ports: ["5432:5432"]
  environment:
    POSTGRES_DB: ${POSTGRES_DB:-realworld}
    POSTGRES_USER: ${POSTGRES_USER:-postgres}
    POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
  volumes:
    - postgres_data:/var/lib/postgresql/data
  healthcheck:
    test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-postgres}"]
    interval: 10s
    timeout: 5s
    retries: 5
```

## Verificações obrigatórias

```bash
# V1 — Validar YAML
docker compose config
# Esperado: YAML válido sem erros

# V2 — Listar serviços
docker compose config --services
# Esperado: app, postgres, prometheus, loki, tempo, grafana

# V3 — Subir stack completa
docker compose up -d
# Aguardar 60s para startup completo

# V4 — Verificar status
docker compose ps
# Esperado: 6 serviços em estado "healthy" ou "running"

# V5 — Health da aplicação
curl -s http://localhost:8080/actuator/health
# Esperado: {"status":"UP"}

# V6 — Endpoint de negócio
curl -s http://localhost:8080/tags
# Esperado: {"tags":[...]}

# V7 — Grafana
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000
# Esperado: 200 ou 302

# V8 — Shutdown
docker compose down
docker compose ps
# Esperado: 0 serviços running
```
