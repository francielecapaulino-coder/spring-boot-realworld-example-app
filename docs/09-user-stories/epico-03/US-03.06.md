## História de usuário

**Como** engenheiro de DevOps que quer garantir que a stack sobe e encerra corretamente a cada execução do CI,
**quero** que exista um script Python `scripts/validate_startup.py` que valide o startup e o shutdown da aplicação via Docker Compose —
**para que** qualquer falha no processo de boot seja detectada automaticamente antes de chegar ao ambiente de desenvolvimento.

---

## Contexto

### Comportamento esperado do script

O script `scripts/validate_startup.py`:

1. Executa `docker compose up -d`
2. Aguarda até **60 segundos** pelo endpoint `/actuator/health` retornar `{"status":"UP"}`
3. Valida que o log de startup contém os campos: `event: "application_startup"`, `timestamp`, `version`, `environment` *(ou valida via `/actuator/health` como alternativa)*
4. Executa `docker compose stop`
5. Aguarda até **30 segundos** confirmando que os containers pararam
6. Retorna **exit code 0** (sucesso) ou **exit code 1** com mensagem descritiva (falha)

### Critério de saída do script

| Condição | Exit code | Mensagem |
|---|---|---|
| `/actuator/health` retorna `{"status":"UP"}` dentro de 60s | 0 | `✓ Startup validated in Xs` |
| Timeout de 60s sem `{"status":"UP"}` | 1 | `✗ Startup validation failed: timeout after 60s` |
| `docker compose up` falha | 1 | `✗ docker compose up failed: <stderr>` |
| Shutdown: containers param dentro de 30s | 0 | `✓ Shutdown validated` |
| Shutdown: timeout de 30s | 1 | `✗ Shutdown validation failed: containers still running` |

## O que esta história faz — e o que NÃO faz

### ✅ O que esta história faz

- Cria `scripts/validate_startup.py` (Python 3, sem dependências externas além de `requests`)
- Script executa `docker compose up -d`, aguarda `/actuator/health` → `UP`, executa `docker compose stop`
- Script retorna exit code 0 (sucesso) ou 1 (falha) com mensagem clara
- Script pode ser executado localmente e no CI (US-03.07)
- Adiciona `requirements-scripts.txt` se necessário (`requests` como única dependência)

### ❌ O que esta história NÃO faz

- Não integra o script ao CI — isso é US-03.07
- Não valida endpoints de negócio além de `/actuator/health` — isso é EPIC-09 (E2E)
- Não monitora a aplicação em produção — fora do escopo

## Estrutura do script

```python
#!/usr/bin/env python3
"""
validate_startup.py — valida startup e shutdown da stack via Docker Compose
Exit code 0: sucesso | Exit code 1: falha com mensagem descritiva
"""

import subprocess
import sys
import time
import requests  # pip install requests

HEALTH_URL = "http://localhost:8080/actuator/health"
STARTUP_TIMEOUT_S = 60
SHUTDOWN_TIMEOUT_S = 30
POLL_INTERVAL_S = 3


def run(cmd: list[str]) -> subprocess.CompletedProcess:
    return subprocess.run(cmd, capture_output=True, text=True)


def wait_for_health(timeout: int) -> bool:
    deadline = time.time() + timeout
    while time.time() < deadline:
        try:
            response = requests.get(HEALTH_URL, timeout=5)
            if response.status_code == 200 and response.json().get("status") == "UP":
                return True
        except Exception:
            pass
        time.sleep(POLL_INTERVAL_S)
    return False


def main() -> int:
    print("→ Starting docker compose...")
    result = run(["docker", "compose", "up", "-d"])
    if result.returncode != 0:
        print(f"✗ docker compose up failed:\n{result.stderr}", file=sys.stderr)
        return 1

    print(f"→ Waiting up to {STARTUP_TIMEOUT_S}s for /actuator/health UP...")
    start = time.time()
    if not wait_for_health(STARTUP_TIMEOUT_S):
        print(f"✗ Startup validation failed: timeout after {STARTUP_TIMEOUT_S}s", file=sys.stderr)
        run(["docker", "compose", "down"])
        return 1

    elapsed = round(time.time() - start)
    print(f"✓ Startup validated in {elapsed}s")

    print("→ Stopping docker compose...")
    run(["docker", "compose", "stop"])
    print("✓ Shutdown validated")
    return 0


if __name__ == "__main__":
    sys.exit(main())
```
