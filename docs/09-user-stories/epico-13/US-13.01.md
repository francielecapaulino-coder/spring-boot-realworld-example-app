## História de usuário

**Como** leitor da plataforma,  
**quero** ver o tempo estimado de leitura em cada artigo,  
**para que** possa decidir se tenho tempo disponível antes de começar a ler.

---

## Contexto

Cada artigo tem um `body` com texto em Markdown. Dividindo o número de palavras por 200 (palavras por minuto — padrão de mercado), obtemos o tempo estimado. O resultado é arredondado para cima com mínimo de 1 minuto. O campo é calculado na criação/edição do artigo e persistido na coluna `reading_time_minutes`. Artigos existentes sem o campo recebem o valor na primeira leitura (lazy update — US-13.02).

**Fórmula:**
```
wordCount   = body.split(/\s+/).length  // palavras separadas por espaço
readingTime = max(1, ceil(wordCount / 200))
```

**Exemplos:**
- 400 palavras → `ceil(400/200)` = 2 min
- 50 palavras → `max(1, ceil(50/200))` = 1 min
- 2 500 palavras → `ceil(2500/200)` = 13 min

* Adicionar testes unitários para `ReadingTimeCalculator`: texto normal, texto vazio, texto de 1 palavra, texto longo

## Regras técnicas

* Contagem de palavras: `body.trim().split("\\s+").length` (evita contar strings vazias)
* Campo `null` em artigos existentes sem cálculo → tratado em US-13.02 (lazy)
* `readingTimeMinutes` **nunca** pode ser negativo ou zero
* Não expor `reading_time_minutes` como snake_case no JSON — usar `readingTimeMinutes` (camelCase)
