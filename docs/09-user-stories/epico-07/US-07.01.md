## História de usuário

**Como** guardião da qualidade da mutação,  
**quero** configurar o plugin Pitest para rodar mutação testing como parte do build regular,  
**para que** tenhamos cobertura ≥ 95% detectando falha de testes sem lobbying pelo código.

---

## Contexto

Pitest executa mutação testing, alterando small chunks bytecode e rodando testes para saber se são pegos. Ele's commonly integrated via Gradle plugin. Neste projeto usaremos Gradle 9.3.1 e já temos JUnit 5. A configuração deve excluir classes geradas (como GraphQL e Lombok), focar em classes de serviço e repositório e reportar 95% como mínimo. Relatório HTML gerado em `build/reports/pitest` para CI inspection.

---

## Escopo

* Adicionar `info.solidsoft.gradle.pitest` plugin ao `build.gradle`
* Configurar `targetClasses` para `io.spring.**` (excluir `.graphql.**` e `*.generated.**`)
* Configurar `targetTests` para `io.spring.**`
* Excluir classes Lombok e classes de teste
* Definir `mutationThreshold = 95`
* Configurar `outputFormats = ['HTML', 'XML']`
* Adicionar tarefa `pitest` que depende de `test`
* Excluir tipo de mutadores `REMOVE_CONDITIONALS` se necessário para manter CI rápido

## Fora de escopo

* Ajuste individual de testes para reach 95% (ser feito em histórias seguintes)
* Integração com GitHub Actions badges (pode ser depois da primeira conecção)
* Cobertura de 100% ou exclusão de classes anemicas

## Critérios de aceite

* [ ] `./gradlew pitest` roda sem falha e gera HTML/XML
* [ ] Relatório HTML mostra 0 KILLED ou 0 tempo com target classes
* [ ] `mutationThreshold = 95` configurado e respeitado
* [ ] Build fail se below 95%, otherwise OK
* [ ] Excluídas: geradores de código GraphQL, Lombok, test/especificados

## Regras técnicas

* Não deletar `test` task; adicionar `pitest` como dependente
* Usar JUnit 5 engine, mas Pitest suporta isso nativamente
* Escopo das mutações fixo ouTempo-level

## Dependências

## Resultado esperado

Plugin Pitest configurado, rodando mutação testing e falhando automaticamente se a cobertura cair abaixo de 95%, com relatório HTML disponível.
