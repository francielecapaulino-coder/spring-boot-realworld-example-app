## História de usuário

**Como** desenvolvedor responsável pela qualidade,  
**quero** escrever testes unitários para os application services,  
**para que** mutações na lógica de orquestração sejam detectadas pelo Pitest.

---

## Contexto

Os application services (`io.spring.application.*`) orquestram repositórios e aplicam regras de negócio como validação de unicidade, autorização e transformação de dados.

**Classes alvo (todas em `io.spring.application.*`):**
- `ArticleCommandService` — criar, atualizar, deletar artigos
- `ArticleQueryService` — listagem, busca por slug, feed
- `UserService` (ou equivalente) — registro, login, atualização de perfil
- `CommentService` — criar, listar, deletar comentários

Testes existentes em `io.spring.application.article.ArticleQueryServiceTest` — verificar antes de criar.

---

## Escopo

* Criar ou completar testes para `io.spring.application.*`
* Usar `@ExtendWith(MockitoExtension.class)` + mock dos repositórios
* Cobrir: cenários de sucesso, exceções lançadas, e casos de borda
* Verificar que o mutation score de `io.spring.application.*` ≥ 95% após os testes

## Fora de escopo

* Testes de infrastructure/repositórios — US-07.04
* Nenhum `@SpringBootTest` nesta história — apenas JUnit 5 + Mockito

## Critérios de aceite

* [ ] Todos os métodos públicos de `ArticleCommandService`, `ArticleQueryService` e `UserService` têm cobertura por mutação
* [ ] Exceções de negócio (artigo não encontrado, usuário já existe) estão cobertas
* [ ] `./gradlew test` 100% verde
* [ ] `./gradlew pitest` — score de `io.spring.application.*` ≥ 95%

## Regras técnicas

* Mocks apenas para dependências externas (repositórios, JWT service)
* Nomenclatura: `should_<comportamento>_when_<condição>()`
* Proibido `@Disabled`

## Dependências

## Resultado esperado

Application services com mutation score ≥ 95%.
