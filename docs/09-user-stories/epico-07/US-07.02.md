## História de usuário

**Como** desenvolvedor responsável pela qualidade,  
**quero** escrever testes unitários abrangentes para as classes do domínio core,  
**para que** o Pitest detecte mutações injetadas nessas classes e o mutation score avance em direção a 95%.

---

## Contexto

O domínio core (`io.spring.core.*`) contém as regras de negócio mais críticas. Classes como `Article`, `User`, `Comment` e `Tag` têm lógica de negócio (validação de slug, update de campos, cálculo de favoritos) que hoje não está adequadamente coberta por testes de mutação.

**Classes alvo (todas em `io.spring.core.*`):**
- `Article` — `update()`, `favorite()`, `unfavorite()`, `isFavorited()`, `getSlug()`, `getFavoritesCount()`
- `User` — `update()`, `isPassword()`, `getEncodedPassword()`, `isFollowing()`
- `Comment` — `getAuthorId()`, `belongsTo()`
- `Tag` — `getName()`
- `ArticleContent` (se existir) — campo de conteúdo do artigo

---

## Escopo

* Criar ou completar testes JUnit 5 para todas as classes de `io.spring.core.*`
* Cada método público da classe deve ter ao menos um teste de sucesso e um de borda/falha
* Usar `assertThat` do AssertJ (já presente no classpath)
* Sem Spring context nos testes — `@ExtendWith(MockitoExtension.class)` onde necessário
* Executar `./gradlew pitest` e verificar se o score das classes `io.spring.core.*` está ≥ 95%

## Fora de escopo

* Testes dos application services — US-07.03
* Testes de infrastructure/repositórios — US-07.04
* Configuração do CI — US-07.05

## Critérios de aceite

* [ ] Cada método público de `Article`, `User`, `Comment`, `Tag` tem ao menos 1 teste
* [ ] `./gradlew test` 100% verde
* [ ] `./gradlew pitest` — mutation score de `io.spring.core.*` ≥ 95%
* [ ] Nenhum teste usa Spring context (`@SpringBootTest`) — apenas JUnit 5 puro

## Regras técnicas

* Usar Mockito apenas para dependências externas — entidades do domínio core não têm dependências Spring
* Nomenclatura: `should_<comportamento>_when_<condição>()` — ex: `should_return_true_when_article_is_favorited()`
* Proibido `@Disabled` — se o teste falha, corrigir o teste ou abrir issue

## Dependências
