## O que é GitAhead e por que usamos neste projeto

GitAhead é um cliente Git visual gratuito (MIT) que mostra o grafo de commits
e branches de forma clara e rápida. Neste projeto, é usado para três finalidades:

1. **Validar commits no branch `bleeding`** — confirmar visualmente que
   `./scripts/bleeding-commit.sh` executou com sucesso

2. **Acompanhar progresso paralelo das duplas** — Dupla A e Dupla B em branches
   separadas enquanto `bleeding` recebe commits de ambas simultaneamente

3. **Auditar Conventional Commits** — verificar antes de abrir uma PR que todos
   os commits seguem o padrão e referenciam a issue correta

> **Status de manutenção:** GitAhead não está mais em desenvolvimento ativo —
> lançamentos de manutenção são feitos conforme necessário. O projeto é estável
> e adequado para este uso. Para Linux ou quem prefere alternativa com
> desenvolvimento ativo, ver seção "Alternativa: Gittyup".

| Atributo | Valor |
|---|---|
| **Versão** | v2.7.1 (dezembro 2023) |
| **Repositório** | https://github.com/gitahead/gitahead |

## Instalação

### macOS (v2.7.1)

**Opção A — Download direto (recomendado):**

```bash
# 1. Baixar o DMG:
# https://github.com/gitahead/gitahead/releases/download/v2.7.1/GitAhead-2.7.1.dmg

# 2. Abrir o arquivo .dmg baixado

# 3. Arrastar GitAhead.app para a pasta Applications

# 4. Primeira abertura — aviso de segurança:
# System Settings > Privacy & Security > clicar "Open Anyway"
# Confirmar "Open" no dialog
```

**Opção B — Homebrew (recomendado para quem já usa Homebrew):**

```bash
brew install --cask gitahead
# Instala v2.7.1 automaticamente
# Requer macOS Big Sur (11) ou posterior
```

**Verificar instalação:**

```bash
# Deve abrir sem erro:
open /Applications/GitAhead.app

# Verificar versão: Menu GitAhead > About GitAhead
# Deve mostrar: Version 2.7.1
```

> **Apple Silicon (M1/M2/M3/M4):** o binário é universal — funciona
> nativamente em chips Apple Silicon sem Rosetta.

> **macOS Sequoia ou Sonoma:** o aviso de segurança pode ter aparência
> diferente. Se `Open Anyway` não aparecer em System Settings:
> ```bash
> sudo xattr -cr /Applications/GitAhead.app
> ```

## Uso no workflow do projeto

### 1. Visualizar o branch `bleeding`

````
1. Painel esquerdo → seção "Branches"

2. Expandir "Remote Branches"

3. Clicar em "origin/bleeding"

4. O histórico mostra os commits automáticos do bleeding-commit.sh:
````

**Formato esperado de cada commit no `bleeding`:**

````
tipo(escopo): step X/Y - descrição da etapa

refs #XX
bleeding-branch-auto-commit: YYYY-MM-DD HH:MM:SS
source-branch: tipo/us-XX.XX-descricao
````

---

### 2. Validar commit após `./scripts/bleeding-commit.sh`

````
1. Pressionar F5 — atualiza com o repositório remoto

2. Selecionar "origin/bleeding" no painel de branches

3. Verificar o commit mais recente:
   ✓ Aparece no topo da lista
   ✓ Mensagem: tipo(escopo): step X/Y - descrição
   ✓ Footer com refs #XX (clicar no commit para expandir)
   ✓ Timestamp corresponde ao momento da execução
   ✓ Diff mostra as mudanças da etapa
````

**Se o commit não aparecer após F5:**

```bash
# Verificar no terminal:
git log origin/bleeding --oneline -5

# Se aparece no terminal mas não no GitAhead:
# Fechar e reabrir o GitAhead

# Se não aparece nem no terminal:
# O script falhou — executar novamente com atenção ao output
```

---

### 3. Acompanhar branches das duplas em paralelo

````
1. Painel esquerdo → selecionar múltiplas branches:
   - master
   - bleeding
   - feat/us-XX.XX-... (Dupla A)
   - test/us-YY.YY-... (Dupla B)

2. O grafo central mostra todas as branches com cores distintas

3. Para ver onde uma branch divergiu da master:
   - Localizar o ponto de bifurcação no grafo

4. Para comparar dois commits:
   - Ctrl+Click em dois commits
   - GitAhead mostra o diff entre eles
````

**Diagrama visual esperado no GitAhead:**

````
● master ─────────────────────────────── (HEAD)
│
├── ● bleeding ── step1/3 ── step2/3 ── step3/3
│                  (Dupla A)   (Dupla B)
│
├──── feat/us-13.01 ── feat/us-13.02 ── (Dupla A)
│
└──── test/us-08.01 ── test/us-08.02 ── (Dupla B)
````

---

### 4. Auditar Conventional Commits antes de abrir PR

````
1. Selecionar a branch de feature no painel esquerdo
   (ex: feat/us-13.02-reading-time-calculation)

2. Commits novos aparecem acima do ponto de bifurcação com master

3. Para cada commit, verificar:
   ✓ Formato: tipo(escopo): descrição em lowercase
   ✓ Tipo válido: feat, fix, chore, docs, test, refactor, ci, perf, style
   ✓ Footer com closes #XX ou refs #XX (expandir o commit)

4. Se algum commit estiver fora do padrão:
   - NÃO abrir a PR
   - Corrigir com: git rebase -i HEAD~N
   - Re-verificar no GitAhead após a correção
````

---

### 5. Verificar que a branch de trabalho foi restaurada após o script

O `bleeding-commit.sh` restaura automaticamente a branch de trabalho.
Confirmar no GitAhead:

````
1. Após executar o script, verificar o indicador HEAD no GitAhead

2. HEAD (seta ou círculo preenchido) deve apontar para sua branch de trabalho
   — NÃO para o bleeding

3. No painel esquerdo: sua branch de feature deve estar em negrito (branch ativa)

4. Se HEAD estiver no bleeding:
   git checkout <sua-branch-de-trabalho>
   Reportar o comportamento inesperado do script
````

## Troubleshooting

### Repositório não aparece na lista inicial

````
Solução: File > Open Repository → navegar até o diretório correto
O diretório deve conter a pasta .git/
Verificar: ls -la ~/projects/spring-boot-realworld-example-app/.git
````

### Branch `bleeding` não aparece no painel de branches

````
Causa: GitAhead não buscou as branches remotas
Solução:
1. Pressionar F5
2. Expandir "Remote Branches" no painel esquerdo
3. Se ainda não aparecer: git fetch origin (no terminal)
````

### Commits não aparecem após `bleeding-commit.sh`

````
Causa 1: GitAhead não atualizou → pressionar F5
Causa 2: Script falhou silenciosamente

Diagnóstico:
git log origin/bleeding --oneline -3
Se aparece no terminal mas não no GitAhead:
  → Fechar e reabrir o GitAhead
Se não aparece nem no terminal:
  → Script falhou — verificar output e executar novamente
````

### GitAhead não abre no macOS (aviso de segurança)

````
macOS Ventura ou anterior:
  System Settings > Privacy & Security > "Open Anyway" > "Open"

macOS Sonoma ou Sequoia:
  Se "Open Anyway" não aparecer, executar no terminal:
  sudo xattr -cr /Applications/GitAhead.app
  Depois tentar abrir novamente
````

### Linux: versão v2.7.1 não disponível

````
Causa: v2.7.0+ não fornece binário pré-compilado para Linux

Soluções:
1. Usar v2.6.3 (última com binário Linux):
   https://github.com/gitahead/gitahead/releases/download/v2.6.3/GitAhead-2.6.3-linux.sh

2. Usar Gittyup (fork mantido com suporte Linux ativo):
   https://github.com/Murmele/Gittyup
   Interface idêntica ao GitAhead
````
